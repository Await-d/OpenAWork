# .agentdocs/workflow/260318-高级能力扩展方案.md

## 任务概览

为跨平台 AI 智能体客户端补充七项高级能力模块，对标业界头部开源项目（LangGraph、Dify、browser-use、CopilotKit、MetaGPT 等）的设计实践。

本方案与前三份方案**完全隔离**，独立时间线，独立交付。

**包含模块**：

1. 多 Agent 编排（可视化执行图 + 角色分工）
2. Token 成本与用量监控
3. 工作流可视化编排（低代码拖拽）
4. 多模态输入（图片/语音/批量文件）
5. CI/CD 代码库深度集成（GitHub 事件触发）
6. Generative UI（生成式交互组件）
7. 浏览器自动化（Playwright/Browser-use 集成）

---

## 模块 V：多 Agent 编排

**设计目标**：支持将复杂任务分解为多个专属 Agent 并发执行，提供可视化执行图实时展示各 Agent 状态与数据流向。

### Agent 角色模型

```typescript
export interface AgentRole {
  id: string;
  name: string; // 'researcher' | 'coder' | 'reviewer' | 'planner'
  description: string;
  systemPrompt: string;
  allowedTools: string[]; // Tool Registry 中的 tool id 白名单
  model?: string; // 可指定不同模型（如 haiku 做路由，sonnet 做执行）
  maxIterations?: number;
}

export interface AgentGraph {
  id: string;
  sessionId: string;
  nodes: AgentNode[];
  edges: AgentEdge[];
  status: 'pending' | 'running' | 'completed' | 'failed';
  startedAt?: number;
  completedAt?: number;
}

export interface AgentNode {
  id: string;
  roleId: string;
  label: string;
  status: 'idle' | 'running' | 'done' | 'error';
  input?: unknown;
  output?: unknown;
  startedAt?: number;
  completedAt?: number;
  position: { x: number; y: number }; // 可视化坐标
}

export interface AgentEdge {
  id: string;
  source: string; // AgentNode.id
  target: string;
  label?: string; // 数据传递说明
  dataType?: string; // 'context' | 'result' | 'tool_output'
}
```

### 编排器接口

```typescript
export interface MultiAgentOrchestrator {
  createGraph(roles: AgentRole[], task: string): Promise<AgentGraph>;
  execute(graphId: string): Promise<void>;
  pause(graphId: string): Promise<void>;
  resume(graphId: string): Promise<void>;
  cancel(graphId: string): Promise<void>;
  getStatus(graphId: string): Promise<AgentGraph>;
  subscribeToEvents(graphId: string, cb: (event: AgentEvent) => void): () => void;
}

export type AgentEvent =
  | { type: 'node_started'; nodeId: string; timestamp: number }
  | { type: 'node_completed'; nodeId: string; output: unknown; timestamp: number }
  | { type: 'node_failed'; nodeId: string; error: string; timestamp: number }
  | { type: 'edge_activated'; edgeId: string; data: unknown; timestamp: number }
  | { type: 'graph_completed'; graphId: string; timestamp: number };
```

### 可视化 UI（React Flow）

- `AgentGraphView`：React Flow 画布，节点 = Agent，边 = 数据流
- 节点颜色：🔵 running / ✅ done / ❌ error / ⬜ idle
- 边动画：数据传递时显示流动粒子效果
- 点击节点：展开该 Agent 的 tool calls + 输出预览
- 移动端：简化为线性列表（角色名 + 状态 + 进度条）

### 内置角色模板

| 角色       | 职责                     | 默认工具              |
| ---------- | ------------------------ | --------------------- |
| Planner    | 任务拆解，生成子任务列表 | 无（纯推理）          |
| Researcher | 信息搜集                 | web-search、file-read |
| Coder      | 代码生成与修改           | file-write、code-exec |
| Reviewer   | 结果审查，输出评分       | file-read             |
| Summarizer | 汇总最终结果             | 无（纯推理）          |

## 模块 W：Token 成本与用量监控

**设计目标**：实时追踪每次 Run、每个 Agent、每个模型的 Token 消耗与成本，提供预算告警与趋势分析。

```typescript
export interface TokenUsageRecord {
  id: string;
  sessionId: string;
  agentId?: string;
  model: string;
  promptTokens: number;
  completionTokens: number;
  totalTokens: number;
  costUsd: number; // 基于模型单价计算
  timestamp: number;
  toolName?: string; // 工具调用耗费（如图像 token）
}

export interface UsageDashboard {
  today: UsageSummary;
  thisWeek: UsageSummary;
  thisMonth: UsageSummary;
  byModel: Record<string, UsageSummary>;
  bySession: SessionUsage[];
  budgetAlert?: BudgetAlert;
}

export interface UsageSummary {
  totalTokens: number;
  totalCostUsd: number;
  sessionCount: number;
  avgCostPerSession: number;
}

export interface BudgetAlert {
  dailyLimitUsd: number;
  monthlyLimitUsd: number;
  currentDailyUsd: number;
  currentMonthlyUsd: number;
  alertThreshold: number; // 0-1，如 0.8 = 80% 时告警
}
```

**模型单价表**（可配置，随 API 价格更新）：

```typescript
export const MODEL_PRICING: Record<string, { inputPer1M: number; outputPer1M: number }> = {
  'claude-opus-4': { inputPer1M: 15, outputPer1M: 75 },
  'claude-sonnet-4': { inputPer1M: 3, outputPer1M: 15 },
  'claude-haiku-3-5': { inputPer1M: 0.8, outputPer1M: 4 },
  'gpt-4o': { inputPer1M: 2.5, outputPer1M: 10 },
  'gemini-2.0-flash': { inputPer1M: 0.1, outputPer1M: 0.4 },
};
```

**UI 组件**：

- `UsageDashboard`：饼图（按模型分布）+ 折线图（每日趋势）+ 总消费卡片
- `SessionCostBadge`：每次 Run 旁显示本次花费（如 $0.023）
- `BudgetAlertBanner`：超过阈值时顶部警告横幅
- `CostEstimator`：发送前估算本次请求预计费用

## 模块 X：工作流可视化编排

**设计目标**：提供低代码拖拽式工作流编辑器，用户可图形化编排 Agent 任务流程，保存为可复用 Workflow Template。

### 节点类型

```typescript
export type WorkflowNodeType =
  | 'trigger' // 触发器（手动/定时/GitHub 事件）
  | 'llm' // LLM 调用节点
  | 'tool' // 工具调用节点
  | 'condition' // 条件分支
  | 'loop' // 循环
  | 'subagent' // 子 Agent 委托
  | 'code' // 自定义代码
  | 'end'; // 结束节点
```
