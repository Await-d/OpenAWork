# .agentdocs/workflow/260318-crush续作特性补充方案.md

## 任务概览

基于对 `charmbracelet/crush`（21K stars，OpenCode 官方续作，Charm 团队维护）的深度分析，识别当前五份方案尚未覆盖的新特性，形成第六份独立补充方案。

**Crush 定位**："Glamourous agentic coding for all" — 对比 OpenCode 更注重跨平台体验、生态开放性、企业级可靠性。

本方案与前五份方案**完全隔离**，独立时间线，独立交付。

---

## Crush 核心新特性分析

### 相比 OpenCode 的主要差异

| 维度          | OpenCode（已归档） | Crush（续作）                               |
| ------------- | ------------------ | ------------------------------------------- |
| 维护团队      | 个人               | Charm 官方团队                              |
| 平台支持      | macOS/Linux        | + Windows/Android/FreeBSD/OpenBSD/NetBSD    |
| Provider 管理 | 静态配置           | Catwalk 在线自动更新                        |
| 模型定价      | 无                 | 内置 cost_per_1m_in/out（每次对话显示成本） |
| Agent Skills  | 基础               | agentskills.io 开放标准                     |
| 文件忽略      | .gitignore         | + .crushignore 独立忽略规则                 |
| 权限          | 基础               | allowed_tools 白名单 + --yolo 全跳过        |
| 工具禁用      | 无                 | disabled_tools 完全隐藏工具                 |
| Commit 归因   | 无                 | Co-Authored-By / Assisted-By trailer        |
| 日志系统      | debug flag         | crush logs --tail/--follow 实时查看         |
| Metrics       | 无                 | 匿名遥测（可 opt-out）                      |
| MCP transport | stdio/SSE          | + HTTP（新增）                              |
| Context 文件  | AGENTS.md          | + CRUSH.md/CLAUDE.md/GEMINI.md              |
| 模型切换      | 重启               | 会话中途切换（保留上下文）                  |
| LSP           | 基础               | filetypes + root_markers 精准匹配           |
| Provider 抽象 | 直接 API           | charm.land/fantasy 统一抽象层               |

## 差距模块设计

### 模块 AA：Catwalk 模型注册中心（在线自动更新）

**设计目标**：Provider 和模型列表从社区维护的在线注册中心自动同步，无需用户手动配置，新模型上线后自动可用。

```typescript
export interface CatwalkProvider {
  id: string;
  name: string;
  type: 'openai' | 'openai-compat' | 'anthropic' | 'gemini' | 'bedrock';
  baseUrl?: string;
  models: CatwalkModel[];
}

export interface CatwalkModel {
  id: string;
  name: string;
  contextWindow: number;
  defaultMaxTokens: number;
  costPer1mIn: number; // 输入 token 每百万成本（USD）
  costPer1mOut: number; // 输出 token 每百万成本（USD）
  costPer1mInCached?: number; // 缓存命中成本
  canReason?: boolean; // 支持推理模式
  supportsAttachments?: boolean;
}

export interface CatwalkRegistry {
  fetchLatest(baseUrl?: string): Promise<CatwalkProvider[]>;
  loadEmbedded(): CatwalkProvider[]; // 内置快照（离线可用）
  loadFromFile(path: string): Promise<CatwalkProvider[]>;
  updateLocal(providers: CatwalkProvider[]): Promise<void>;
  isAutoUpdateEnabled(): boolean;
}
```

**自动更新策略**：

- 应用启动时后台静默检查更新（不阻塞 UI）
- 离线环境：fallback 到内置快照
- 用户可配置 `disable_provider_auto_update: true` 关闭
- 支持自定义 Catwalk 源 URL（私有化部署）

**UI 组件**：

- `ProviderUpdateBadge`：模型列表页显示「已更新 N 个模型」提示
- `ModelCostDisplay`：每个模型旁显示输入/输出单价
- `OfflineModeBanner`：离线时显示「使用内置快照」提示

**包位置**：`packages/catwalk-registry/`

### 模块 AB：会话中途模型切换

**设计目标**：用户可在对话进行中切换 LLM 模型，上下文完整保留，无需重开会话。

```typescript
export interface ModelSwitchManager {
  getCurrentModel(): ModelInfo;
  listAvailableModels(): ModelInfo[];
  switchModel(modelId: string, options?: SwitchOptions): Promise<void>;
  getContextTransferStatus(): ContextTransferStatus;
}

export interface SwitchOptions {
  preserveHistory: boolean; // 默认 true
  compressBeforeSwitch?: boolean; // 超大上下文时先压缩
  reason?: string; // 记录切换原因（审计）
}

export interface ContextTransferStatus {
  tokenCount: number;
  estimatedCostUSD: number; // 转移上下文的预估成本
  compatible: boolean; // 新模型是否支持当前上下文长度
  warning?: string; // 如超出新模型 context window
}
```

**切换场景**：

- 复杂推理任务 → 切到 claude-opus（提升质量）
- 简单任务 → 切到 claude-haiku（节省成本）
- 当前模型配额耗尽 → 自动建议切换备用模型
- 会话中发现新模型上线 → 实时切换

**UI**：命令面板 `Ctrl+O` 打开模型选择器，切换后显示 Toast 确认，显示上下文转移成本估算。

### 模块 AC：多平台原生权限系统

**设计目标**：精细化工具权限控制，支持 allowed_tools 白名单、disabled_tools 完全隐藏、--yolo 全跳过模式，并适配 Windows/Android 平台差异。

```typescript
export interface PermissionConfig {
  allowedTools: string[]; // 白名单内工具无需每次确认
  disabledTools: string[]; // 完全隐藏，LLM 不可见
  yoloMode: boolean; // 跳过所有权限提示（慎用）
  sessionGranted: Map<string, PermissionLevel>;
}

export type PermissionLevel =
  | 'once' // 仅本次
  | 'session' // 本会话
  | 'always' // 永久（写入配置）
  | 'disabled'; // 完全禁用

export interface PermissionManager {
  check(toolName: string, context: ToolContext): Promise<PermissionResult>;
  grant(toolName: string, level: PermissionLevel): void;
  revoke(toolName: string): void;
  isDisabled(toolName: string): boolean;
  getAllowed(): string[];
}
```

**Yolo 模式**：`--yolo` 标志或配置 `yolo_mode: true`，跳过所有权限提示。高风险操作额外警告，日志强制记录。

**disabled_tools 语义**：与 allowed_tools 不同，disabled_tools 从工具列表中**完全移除**，LLM 根本不知道该工具存在，防止 prompt injection 诱导调用。

**平台差异**：

- Windows：使用 Windows Credential Manager 存储永久权限
- Android：使用 Android Keystore
- 移动端：不支持 yolo 模式（安全限制）

### 模块 AD：.crushignore 文件忽略系统

**设计目标**：独立于 .gitignore 的 AI 感知文件过滤层，精确控制哪些文件对 Agent 不可见。

```typescript
export interface CrushIgnoreManager {
  loadIgnoreRules(projectRoot: string): Promise<IgnoreRuleSet>;
  shouldIgnore(filePath: string): boolean;
  listIgnoredFiles(dir: string): Promise<string[]>;
  addRule(pattern: string, scope: 'project' | 'user'): Promise<void>;
}

export interface IgnoreRuleSet {
  gitignoreRules: string[];
  crushignoreRules: string[];
  userRules: string[]; // ~/.config/crush/.crushignore
  effectiveIgnored: string[]; // 实际忽略列表
}
```

**应用场景**：

- `.env` 文件（在 git 中但不应让 AI 看到内容）
- `node_modules/`（git 忽略但 AI 也应忽略）
- 大型二进制文件（避免 token 浪费）
- 敏感业务逻辑文件（合规需求）

**UI**：设置页「文件过滤」tab，显示当前忽略规则，支持添加/删除，预览哪些文件被过滤。

---

### 模块 AE：Git Commit 归因系统

**设计目标**：AI 辅助生成的 Commit 和 PR 自动添加标准化归因 trailer，满足开源合规和审计需求。

```typescript
export type AttributionStyle =
  | 'assisted-by' // Assisted-by: Claude Sonnet 4 via MyApp <app@example.com>
  | 'co-authored-by' // Co-Authored-By: AI <ai@example.com>
  | 'none'; // 不添加归因

export interface AttributionConfig {
  trailerStyle: AttributionStyle;
  generatedWith: boolean; // 添加「Generated with [App]」标记
  customEmail?: string; // 自定义归因邮箱
  includeModelName: boolean; // 在 trailer 中包含模型名称
}

export interface AttributionManager {
  generateTrailer(model: ModelInfo, style: AttributionStyle): string;
  applyToCommit(message: string, config: AttributionConfig): string;
  applyToPR(body: string, config: AttributionConfig): string;
}
```

**Trailer 示例**：`Assisted-by: Claude Sonnet 4 via MyApp <app@example.com>`

### 模块 AF：实时日志系统

```typescript
export interface LogManager {
  tail(lines: number): Promise<LogEntry[]>;
  follow(callback: (entry: LogEntry) => void): () => void;
  filter(level: LogLevel, pattern?: string): Promise<LogEntry[]>;
  getPath(): string;
}

export interface LogEntry {
  timestamp: number;
  level: 'debug' | 'info' | 'warn' | 'error';
  component: 'agent' | 'tool' | 'mcp' | 'lsp' | 'ui';
  message: string;
  metadata?: Record<string, unknown>;
}
```

存储路径：`.crush/logs/crush.log`，UI：`LogViewer`（实时滚动 + 级别过滤）+ `LogExport`。
