# Mock 数据真实化方案

## Task Overview

将 Web 端 6 个页面及 SettingsPage 中使用的硬编码 mock 数据全部替换为真实 Gateway API 调用。

## Complexity Assessment

- 原子步骤：10+ → +2
- 并行流：前端页面独立/后端路由独立 → +2
- 涉及模块：web + agent-gateway + db.ts → +1
- 单步 > 5 min → +1
- 持久化审阅 → +1
- OpenCode 可用 → -1
- **Total: +6 → Full orchestration**
- **Routing rationale**: 6页面 mock→API，需新建 Gateway 路由、扩展 DB schema、前端接线，多流并行。

## Solution Design

### 架构思路

1. **Gateway 新增路由文件** `services/agent-gateway/src/routes/`，按域拆分：
   - `usage.ts` — 用量统计 + 费用分解
   - `team.ts` — 团队成员/任务/消息
   - `channels.ts` — 渠道 CRUD（已有 OC channel 实现，接线即可）
   - `schedules.ts` — Cron Job CRUD（已有 CronScheduler，接线即可）
   - `workflows.ts` — 工作流模板 CRUD
   - `settings.ts` — MCP状态/权限决策/诊断/Worker/模型定价

2. **DB 扩展**（`db.ts` migrate()）：
   - `usage_records` — 月度用量记录
   - `team_members` / `team_tasks` / `team_messages` — 团队协作
   - `workflow_templates` — 已存在于 multi-agent，迁移 SQLite schema

3. **前端接入** — 各页面用 `fetch`/`useEffect` 替换 `useState(MOCK_*)`
