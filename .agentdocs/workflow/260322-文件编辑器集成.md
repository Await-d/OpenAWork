# 文件编辑器集成实施方案

## 任务概述

在 Chat 页面集成 Monaco Editor，实现 VSCode 风格的文件预览/编辑体验：
- 左侧文件树（含文件图标）
- 主区域左右分割（对话区 | 编辑器区）
- 文件标签页（多文件同时打开）
- 文件路径面包屑
- diff 视图
- 保存到服务器（后端文件写入接口）

## 复杂度评估

- 原子步骤：10 → +2
- 并行流：后端 + 前端可并行 → +2
- 模块数：5 → +1
- 单步 >5min → +1
- 持久化 → +1
- OpenCode → -1
- **总分：6 → Full orchestration（Mode A）**

## 现状分析

### 后端（已有）
- `GET /workspace/file?path=...` — 读取文件内容（最大 512KB）
- `GET /workspace/tree` — 目录树
- `GET /workspace/review/status` — git 改动列表
- `GET /workspace/review/diff` — 单文件 diff
- **缺少**：`PUT /workspace/file` — 文件写入

### 前端（已有）
- `FileTreePanel`（shared-ui）— 含 git 状态标记，可复用
- `fetchTree`、`fetchRootPath` — useWorkspace hook 已有
- `WorkspacePickerModal` 中的文件树 + `FileTreeView` 组件
- 左侧侧边栏已有「文件树」Tab，可作为文件选择入口

### 缺少
- Monaco Editor 依赖（`@monaco-editor/react`）
- 文件写入后端接口
- 主区域分割布局（拖动条）
- 文件标签页管理
- 文件图标映射
- 面包屑导航组件

## 解决方案设计

### 架构决策
1. Monaco 以 `@monaco-editor/react` 封装，懒加载（React.lazy）避免首屏体积
2. 主区域布局：ChatPage 主内容区新增「编辑器模式」切换，通过 CSS flex + 拖动条实现左右分割
3. 文件状态集中在 `useFileEditor` hook 管理（打开文件列表、当前激活标签、脏状态）
4. 文件图标：基于扩展名的纯 SVG/emoji 映射（不引入额外图标库）
5. 后端写入接口：`PUT /workspace/file`，body 为 `{ path, content }`

## 风险

- Monaco 包体积较大（~2MB gzip 后约 300KB），通过 React.lazy 懒加载缓解
- 文件写入需严格路径安全校验，防止路径穿越
- 大文件（>512KB）只读展示，不允许编辑
