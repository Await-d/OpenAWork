# 手机端功能补全方案

## 任务概述

修复手机端 P0 Bug 并接通已存在但未挂载的 UI 组件，使手机端达到基本可用状态。

## 复杂度评估

- 原子步骤：8 步 → +2
- 独立并行流：部分可并行 → +2

- 单步 >5 min：是 → +1
- 持久化产物：是 → +1
- OpenCode 可用：是 → -1
- **总分：6**

## 当前缺口清单

### P0 Bug（影响可用性）

| 编号 | 位置 | 问题 | 影响 |
|------|------|------|------|
| B-01 | `AppNavigator.tsx:49` | 无 token 时跳 `sessions` 而非 `onboarding` | 未登录用户进入主页面后所有 API 401 |
| B-02 | `CloudLoginStep` / `ClientLoginStep` | 登录成功后未写入 `TOKEN_EXPIRES_AT_KEY` 到 SecureStore | token 自动刷新永远不触发 |

| B-04 | `HostDoneStep` | Host 模式完成后未写入任何 token/gatewayUrl | Host 模式引导完成后无法登录 |
| B-05 | `ChatScreen` | 进入聊天页不加载历史消息 | 每次聊天都是空白 |

## 实施计划

### Phase 1：P0 Bug 修复（不可跳过）

- [x] B-01：`AppNavigator.tsx` — 无 token 且非 loading/onboarding 时跳 `onboarding` 而非 `sessions`
- [x] B-02：`CloudLoginStep` / `ClientLoginStep` — 登录成功后 `await SecureStore.setItemAsync(TOKEN_EXPIRES_AT_KEY, String(Date.now() + expiresInMs))`

### Phase 2：ChatScreen 历史加载（P0）

- [x] B-05：`ChatScreen` 挂载时通过 `createSessionsClient(gatewayUrl).get(token, sessionId)` 拉取历史，渲染到 messages 列表

- [x] C-01：`AgentActivityPanel` — 解析 WS 流事件中的 `tool_call`/`tool_result` 类型，传入 activities 状态

- [x] C-04：`DialogueModeSelector` — 挂载到 ChatScreen 顶部

## 注意事项

- `TOKEN_EXPIRES_AT_KEY = 'openwork_token_expires_at'` 目前仍在 `AppNavigator.tsx` 与 `OnboardingWizard.tsx` 间通过字符串字面量复用，后续可提取为共享常量
- `ChatScreen` 已使用 `createSessionsClient(gatewayUrl).get(token, sessionId)` 回填历史；如果后端后续暴露独立消息端点，可再切换为更轻量的 messages API
- Host 模式已新增 `hostConfig.gatewayUrl` 字段、`host-gateway` 步骤与 `host-login` 步骤
- `MobileVoiceRecorder.stopRecording` 当前仍返回 `'[语音输入]'`，先完成 UI 闭环；真实 STT（expo-av + Whisper）保留为下一阶段
- `MobileAttachmentBar` 当前使用 placeholder 附件按钮，尚未接入 `expo-document-picker` / `expo-image-picker`
- `AgentActivityPanel` 当前已展示 tool start/result；若后端后续开放 task/permission/question 更多 WS 事件，可继续扩展为完整运行轨迹面板
