# 侧边栏与右侧面板扩展实施方案

## 任务概述

对 Chat 页面的左侧侧边栏和右侧面板进行扩展，增加会话信息、上下文、MCP、LSP 等展示模块，同时强化左侧侧边栏的功能密度。

## 复杂度评估

- 原子步骤：8步 → +2
- 并行流：可并行（前后端无依赖） → +2
- 涉及模块：Layout.tsx / ChatPage.tsx / shared-ui / agent-gateway（4个） → +1
- 单步预计时间 >5 min → +1
- 结果需持久化 → +1
- OpenCode 可用 → -1
- **总分：6**
- **选择模式：Full orchestration（Mode A，直接 task() 分发）**
- **理由：**多文件、多模块、步骤间有依赖，但可按区域并行，适合 Mode A 直接分发子任务。

## 现状分析（已确认无重复）

### 左侧侧边栏现有内容
- 会话列表（按工作区分组）
- 新建会话 + 工作区选择按钮

### 右侧面板现有 Tab（4个）
| Tab | 内容 |
|-----|------|
| 计划 | PlanPanel — Agent todo 任务列表 |
| 工具 | ToolCallCard — 所有工具调用入/出参 |
| 可视化 | AgentDAGGraph + AgentVizPanel |
| 历史 | PlanHistoryPanel |

## 实施计划

### 阶段一：左侧侧边栏扩展（并行可行）

- [ ] T-01: 左侧侧边栏 — 搜索栏  
  在操作栏下方新增会话搜索输入框，实时过滤会话列表

- [ ] T-02: 左侧侧边栏 — Tab 切换（会话 | 文件树）  
  顶部增加两个 Tab：「会话」（现有内容）+ 「文件树」（基于 fetchTree API 展示当前工作区目录）

- [ ] T-03: 左侧侧边栏 — 工作区 git 状态行  
  每个工作区分组标题下方显示：git 分支、未提交改动数，调用 `/workspace/review/status` 接口

### 阶段二：右侧面板新增 Tab（依赖后端接口确认）

- [ ] T-04: 右侧面板 — `信息` Tab  
  展示当前会话 metadata：会话ID、创建时间、消息数、工作区路径、对话模式、YOLO模式、token 估算

- [ ] T-05: 右侧面板 — `上下文` Tab  
  展示系统注入内容列表：工作区路径注入、模式 prompt 前缀、@mention 文件列表（来自 attachmentItems）、token 占比卡片

- [ ] T-06: 右侧面板 — `MCP` Tab  
  展示当前激活的 MCP 服务器列表及状态（已连接/未激活），每个服务器展开可查看工具列表。调用 `/mcp/servers` 接口。

- [ ] T-07: 右侧面板 — 工具 Tab 筛选器  
  在现有「工具」Tab 顶部增加类型筛选（全部 / LSP / 文件 / 网络 / 其他），LSP 工具调用单独聚合视图

### 阶段三：整合与验证

- [ ] T-08: LSP 诊断通过验证，E2E 冒烟测试

## 依赖关系 DAG

```
T-01 ──┐
T-02 ──┤→ T-08（验证）
T-03 ──┘
T-04 ──┐
T-05 ──┤→ T-08
```

T-01~T-07 可并行，T-08 依赖所有前置完成。

## 备注

- MCP 接口需确认后端是否已有 `/mcp/servers` 路由，若无则 T-06 需同步新增
- 文件树 Tab 复用 WorkspacePickerModal 的 fetchTree，不新增接口
- token 估算为前端本地计算（字符数 / 4 近似），无需后端
