# .agentdocs/workflow/260319-lsp集成方案.md

## 任务概览

为跨平台 AI 智能体系统设计并实现 LSP（Language Server Protocol）集成层。
参考 anomalyco/opencode（125K stars）的成熟 LSP 实现，移植并适配到我们的
Monorepo 架构（Tauri 桌面端 + Expo 移动端 + Fastify Agent Gateway）。

**目标**：Agent 在执行代码修改后能获取实时诊断（错误/警告），支持 hover、
definition、references 等语义操作，提升代码生成质量。

- [anomalyco/opencode](https://github.com/anomalyco/opencode)（`packages/opencode/src/lsp/`）

## 架构设计

### 平台差异与 LSP 部署策略

```
桌面端（Tauri）
  └─ 本地 spawn LSP 服务器进程（完整能力）
  └─ 直接读取本地文件系统
  └─ 通过 WebSocket 向 Agent Gateway 推送诊断结果

移动端（Expo）
  └─ 不直接 spawn LSP（无本地进程权限）
  └─ 通过 Agent Gateway WebSocket 订阅桌面端推送的诊断结果
  └─ 只读模式：获取诊断、hover 信息

Agent Gateway（Fastify）
  └─ 接收桌面端 LSP 诊断，广播给所有连接的客户端
  └─ 提供 REST API：/lsp/diagnostics、/lsp/status
  └─ Agent 工具调用层消费 LSP 诊断（代码修改后自动验证）
```

### 1. LSP 服务器定义（参考 OpenCode server.ts）

```typescript
// packages/lsp-client/src/server.ts

export interface LSPServerHandle {
  process: ChildProcessWithoutNullStreams;
  initialization?: Record<string, unknown>;
}

export interface LSPServerInfo {
  id: string;
  extensions: string[];           // 支持的文件扩展名
  root: RootFunction;             // Monorepo 感知的 root 定位
  spawn(root: string): Promise<LSPServerHandle | undefined>;
}

// root 定位辅助：向上遍历找最近匹配的目录（支持 exclude 处理 Monorepo 边界）
const NearestRoot = (
  includePatterns: string[],
  excludePatterns?: string[]
): RootFunction => async (file) => {
  // 遍历 file 所在目录向上，找到第一个包含 includePatterns 的目录
  // 遇到 excludePatterns 中的文件时返回 undefined（避免越过边界）
  ...
}

// MVP 阶段支持的服务器
export const TypescriptServer: LSPServerInfo = {
  id: 'typescript',
  extensions: ['.ts', '.tsx', '.js', '.jsx', '.mjs', '.cjs', '.mts', '.cts'],
  root: NearestRoot(
    ['package-lock.json', 'bun.lockb', 'bun.lock', 'pnpm-lock.yaml', 'yarn.lock'],
    ['deno.json', 'deno.jsonc'] // Deno 项目排除
  ),
  async spawn(root) {
    const tsserver = resolveModule('typescript/lib/tsserver.js', root);
    if (!tsserver) return;
    return {
      process: spawn('typescript-language-server', ['--stdio'], { cwd: root }),
      initialization: { tsserver: { path: tsserver } },
    };
  },
};

export const GoplsServer: LSPServerInfo = {
  id: 'gopls',
  extensions: ['.go'],
  root: NearestRoot(['go.mod', 'go.sum', 'go.work']),
  async spawn(root) {
    const bin = which('gopls');
    if (!bin) return; // 提示用户安装
    return { process: spawn(bin, { cwd: root }) };
  },
};

export const PyrightServer: LSPServerInfo = {
  id: 'pyright',
  extensions: ['.py', '.pyi'],
  root: NearestRoot(['pyproject.toml', 'setup.py', 'requirements.txt', 'Pipfile']),
  async spawn(root) {
    // 优先本地，自动安装回退
    const bin = which('pyright-langserver') ?? await autoInstall('pyright');
    if (!bin) return;
    return { process: spawn(bin, ['--stdio'], { cwd: root }) };
  },
};
```

### 2. LSP 客户端（参考 OpenCode client.ts）

```typescript
// packages/lsp-client/src/client.ts
import { createMessageConnection, StreamMessageReader, StreamMessageWriter } from 'vscode-jsonrpc/node';
import type { Diagnostic } from 'vscode-languageserver-types';

const DIAGNOSTICS_DEBOUNCE_MS = 150; // 防 LSP 多次推送

export interface LSPClientInfo {
  serverID: string;
  root: string;
  connection: MessageConnection;
  diagnostics: Map<string, Diagnostic[]>;
  notify: {
    open(input: { path: string }): Promise<void>;
  };
  waitForDiagnostics(input: { path: string }): Promise<void>;
  hover(input: { file: string; line: number; character: number }): Promise<unknown>;
  definition(input: { file: string; line: number; character: number }): Promise<unknown[]>;
  references(input: { file: string; line: number; character: number }): Promise<unknown[]>;
  shutdown(): Promise<void>;
}

export async function createLSPClient(input: {
  serverID: string;
  server: LSPServerHandle;
  root: string;
  onDiagnostics?: (path: string, diagnostics: Diagnostic[]) => void;
}): Promise<LSPClientInfo> {
  const connection = createMessageConnection(
    new StreamMessageReader(input.server.process.stdout as any),
    new StreamMessageWriter(input.server.process.stdin as any),
  );

  const diagnostics = new Map<string, Diagnostic[]>();

  // 接收诊断推送（带去抖动）
  connection.onNotification('textDocument/publishDiagnostics', (params) => {
    const filePath = normalizeFilePath(fileURLToPath(params.uri));
    diagnostics.set(filePath, params.diagnostics);
    // 去抖动：等待 LSP 推送完整诊断（语法诊断先到，语义诊断后到）
    debouncedNotify(filePath, params.diagnostics, input.onDiagnostics);
  });

  // 处理服务器请求
  connection.onRequest('workspace/configuration', async () => [input.server.initialization ?? {}]);
  connection.onRequest('client/registerCapability', async () => {});
  connection.onRequest('workspace/workspaceFolders', async () => [
    { name: 'workspace', uri: pathToFileURL(input.root).href },
  ]);
  connection.listen();

  // 发送 initialize
  await withTimeout(
    connection.sendRequest('initialize', buildInitializeParams(input.root, input.server.initialization)),
    45_000,
  );
  await connection.sendNotification('initialized', {});

  return {
    serverID: input.serverID,
    root: input.root,
    connection,
    diagnostics,
    notify: {
      async open({ path: filePath }) {
        const text = await fs.readFile(filePath, 'utf-8');
        const languageId = LANGUAGE_EXTENSIONS[path.extname(filePath)] ?? 'plaintext';
        await connection.sendNotification('textDocument/didOpen', {
          textDocument: { uri: pathToFileURL(filePath).href, languageId, version: 0, text },
        });
      },
    },
    async waitForDiagnostics({ path: filePath }) {
      // 等待该文件的诊断推送（带 3s 超时）
      ...
    },
    async hover({ file, line, character }) {
      return connection.sendRequest('textDocument/hover', {
        textDocument: { uri: pathToFileURL(file).href },
        position: { line, character },
      }).catch(() => null);
    },
    async definition({ file, line, character }) {
      return connection.sendRequest('textDocument/definition', {
        textDocument: { uri: pathToFileURL(file).href },
        position: { line, character },
      }).catch(() => []);
    },
    async references({ file, line, character }) {
      return connection.sendRequest('textDocument/references', {
        textDocument: { uri: pathToFileURL(file).href },
        position: { line, character },
        context: { includeDeclaration: true },
      }).catch(() => []);
    },
    async shutdown() {
      connection.end();
      connection.dispose();
      input.server.process.kill();
    },
  };
}
```

### 3. 门面层（统一 API）

```typescript
// packages/lsp-client/src/index.ts

export class LSPManager {
  private clients: LSPClientInfo[] = [];
  private servers: Record<string, LSPServerInfo>;
  private spawning = new Map<string, Promise<LSPClientInfo | undefined>>();
  private broken = new Set<string>();

  constructor(servers: LSPServerInfo[]) {
    this.servers = Object.fromEntries(servers.map(s => [s.id, s]));
  }

  // 打开文件，触发 LSP 服务器启动（如未启动）并等待诊断
  async touchFile(filePath: string, waitForDiagnostics = false): Promise<void>;

  // 获取所有文件的当前诊断
  async diagnostics(): Promise<Record<string, Diagnostic[]>>;

  // 获取指定位置的悬浮信息
  async hover(input: { file: string; line: number; character: number }): Promise<unknown>;

  // 跳转到定义
  async definition(input: { file: string; line: number; character: number }): Promise<unknown[]>;

  // 查找引用
  async references(input: { file: string; line: number; character: number }): Promise<unknown[]>;

  // 获取当前所有 LSP 服务器状态
  async status(): Promise<LSPServerStatus[]>;

  // 关闭所有客户端
  async shutdown(): Promise<void>;
}
```

### 4. Agent Gateway LSP 代理路由

```typescript
// services/agent-gateway/src/lsp/router.ts

// GET /lsp/status — 返回所有已连接 LSP 服务器状态
fastify.get('/lsp/status', async () => lspManager.status());

// GET /lsp/diagnostics — 返回当前所有文件诊断（Agent 工具调用时消费）
fastify.get('/lsp/diagnostics', async () => lspManager.diagnostics());

// POST /lsp/touch — Agent 修改文件后通知 LSP 更新
fastify.post('/lsp/touch', async (req) => {
  const { path, waitForDiagnostics } = req.body;
  await lspManager.touchFile(path, waitForDiagnostics);
  return { ok: true };
});

// WS /lsp/events — 实时推送诊断变化给前端（移动端订阅）
fastify.get('/lsp/events', { websocket: true }, (socket) => {
  const unsub = lspManager.onDiagnosticsUpdate((path, diagnostics) => {
    socket.send(JSON.stringify({ type: 'diagnostics', path, diagnostics }));
  });
  socket.on('close', unsub);
});
```

### 5. Agent 工具集成（代码修改后自动验证）

```typescript
// packages/agent-core/src/tools/lsp-diagnostics.ts

// Agent 修改文件后自动调用此工具获取诊断
export const lspDiagnosticsTool = {
  name: 'lsp_diagnostics',
  description: '获取指定文件的 LSP 诊断（错误/警告），用于验证代码修改是否引入新错误',
  parameters: z.object({
    filePaths: z.array(z.string()).describe('需要检查的文件路径列表'),
  }),
  async execute({ filePaths }) {
    // 通知 LSP 更新文件
    await Promise.all(filePaths.map(p => lspClient.touchFile(p, true)));
    const all = await lspClient.diagnostics();
    // 只返回请求文件的诊断
    return filePaths.reduce((acc, p) => {
      acc[p] = (all[p] ?? []).map(d => ({
        severity: ['', 'ERROR', 'WARN', 'INFO', 'HINT'][d.severity ?? 1],
        line: d.range.start.line + 1,
        col: d.range.start.character + 1,
        message: d.message,
        source: d.source,
      }));
      return acc;
    }, {} as Record<string, unknown[]>);
  },
};
```

## 与 OpenCode 的关键差异适配

| 差异点 | OpenCode 方案 | 我们的适配 |
|--------|-------------|----------|
| 运行时 | Bun | Node.js（`vscode-jsonrpc/node` 不变）|
| 状态管理 | `Instance.state()`（单进程） | `LSPManager` 类（可注入 Gateway 服务）|
| 文件读取 | Bun 原生 API | Node.js `fs/promises` |
| 自动下载 | `BunProc.which()` | `npm exec` / `npx` 回退 |
| LSP 服务器 | 20+ 语言（完整） | MVP: TS + Go + Python（后续扩展）|
| 平台 | 本地 TUI | 桌面端 spawn，移动端代理订阅 |
| 事件总线 | 内部 `Bus` | `EventEmitter` / Gateway WS 广播 |

## 风险矩阵

| 风险 | 概率 | 影响 | 控制措施 |
|------|------|------|----------|
| LSP 服务器启动慢（>10s）| 中 | 中 | 懒加载 + 后台预热；首次启动提示 loading |
| 不同语言服务器 API 不一致 | 高 | 低 | 统一封装，catch 所有请求错误返回空值 |
| Monorepo 中 root 定位错误 | 中 | 高 | NearestRoot exclude 机制 + 集成测试覆盖 |
| 移动端 WS 断线导致诊断丢失 | 中 | 低 | 断线重连后拉取全量诊断快照 |
| LSP 内存占用过高（多实例）| 中 | 中 | broken 集合防止反复 spawn；idle 超时关闭 |

---

## 度量指标

- LSP 诊断首次响应时间（文件打开 → 首条诊断，目标 P50 < 5s）
- 诊断准确率（LSP 报告的错误与实际编译错误吻合率）
- Agent 代码修改后诊断验证覆盖率（目标 100% 修改文件触发验证）
- LSP 进程内存占用（单实例目标 < 200MB）
- 移动端诊断推送延迟（从桌面端发出 → 移动端收到，目标 < 500ms）
