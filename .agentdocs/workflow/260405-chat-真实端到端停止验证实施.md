# 260405 Chat 真实端到端停止验证实施

## Task Overview

在真实 gateway 驱动的刷新恢复 E2E 已打通后，继续补上“刷新后 attach 恢复成功时，当前页仍可直接停止并正确收口到已停止状态”的真实浏览器验证。

## Current Analysis

- 当前真实 E2E 已覆盖：发送消息 → 第一段流文本出现 → 刷新 → attach 恢复 → 最终消息落定。
- 仍缺一个真实控制边界：刷新后虽然恢复了流，但当前页是否还能真正 stop，并把 UI 收口到“已停止”。

## Solution Design

- 复用现有 `chat-stream-attach-live.spec.ts` 与 live config；
- 新增第二条真实 E2E：
  - 发送消息
  - 第一段流文本出现后刷新
  - 刷新后确认“当前运行流仍受此页控制”与停止按钮可见
  - 点击停止
  - 断言 UI 收口到“已停止”，并且后续第二段文本不再继续出现

## Complexity Assessment

- Atomic steps: 3–4 → 0
- Parallel streams: real e2e + verification + docs sync → +2
- Modules/systems/services: web + gateway + mock upstream → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 4
- **Chosen mode**: Full orchestration
- **Routing rationale**: 尽管主要是单条 E2E 扩展，但它依赖真实多进程链路并需要记录验收边界，适合作为独立工作流沉淀。
