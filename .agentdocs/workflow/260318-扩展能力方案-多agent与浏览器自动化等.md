# .agentdocs/workflow/260318-扩展能力方案-多agent与浏览器自动化等.md

## 任务概览

基于用户选择的 7 个扩展方向，设计第四份独立补充方案，覆盖：

1. 多 Agent 编排（可视化执行图 + 角色分工）
2. 浏览器自动化（Playwright/Browser-use 集成）
3. Token 成本与用量监控
4. 工作流可视化编排（低代码拖拽）
5. 多模态输入（图片/语音/批量文件）
6. CI/CD 代码库深度集成（GitHub 事件触发）
7. Generative UI（生成式交互组件）

本方案与前三份方案**完全隔离**，独立时间线，独立交付。

## 模块设计

### 模块 V：多 Agent 编排（Multi-Agent Orchestration）

**设计目标**：支持主 Agent 将子任务委托给专业子 Agent，形成可观测的执行 DAG，用户可在 UI 中看到 Agent 角色、任务分配、执行状态。

**参考**：LangGraph（状态机图）、MetaGPT（角色分工）、bytedance/deer-flow（SuperAgent + subagents）。

#### Agent 角色定义

```typescript
export interface AgentRole {

  allowedTools: string[]; // 该角色可调用的工具列表
```

#### 补充：决策层协议（Decision Layer Protocol）

> 参考 HelloAGENTS 路由机制，现有设计覆盖了「执行层」（如何分发任务、DAG 调度），但缺少「决策层」（是否需要多 Agent、信息是否充分、用户控制权、失败升级策略）。以下四个接口补全这一缺口。

##### 1. 对话路由层（补入 `packages/agent-core`）

在对话入口判断任务复杂度，决定是否需要拉起多 Agent 协作：

```typescript
// packages/agent-core/src/routing.ts

export type RoutingLevel =
  | 'R0' // 直接回答，无需执行（问答/解释）
  | 'R1' // 单 Agent 直接执行（目标明确、单点修改）
  | 'R2' // 多 Agent 简化流（需分析后定位，局部决策）
  | 'R3'; // 多 Agent 完整流（架构级、多方案、跨模块）

export interface RoutingDimensions {
  needsAction: boolean;        // 是否需要执行操作（否→R0）
  targetClear: boolean;        // 目标是否可直接定位（是→倾向R1）
  decisionScope: 'none' | 'local' | 'architectural'; // 决策范围
  impactScope: 'single' | 'multi' | 'cross-system';  // 影响范围
  riskLevel: 'safe' | 'moderate' | 'high';           // 风险等级（high→强制R3）
}

export interface RoutingDecision {
  level: RoutingLevel;
  dimensions: RoutingDimensions;
  clarifications?: ClarificationQuestion[]; // R2/R3 时，信息不充分则追问
  reason: string;
}

export interface ClarificationQuestion {
  dimension: 'goal' | 'constraint' | 'deliverable' | 'acceptance'; // 信息依赖链顺序
  question: string;
  options?: string[]; // 2-4 个选项，最后一个可为「自由输入」
}

export interface ConversationRouter {
  /**
   * 评估输入，返回路由决策。
   * - R0/R1: 直接返回，调用方可立即执行
   * - R2/R3: 若 clarifications 非空，需先向用户追问再执行
   */
  evaluate(input: string, context: SessionContext): Promise<RoutingDecision>;
}
```

**信息依赖链**（追问顺序，不可乱序）：
```
目标范围(goal) → 约束条件(constraint) → 成果规格(deliverable) → 验收标准(acceptance)
```
每次只追问链上当前最缺的那个维度，用户回复后重新评估，最多 3 轮。

##### 2. 执行模式（补入 `MultiAgentOrchestrator`）

用户可在 R2/R3 确认时选择执行自主程度：

```typescript
// packages/multi-agent/src/orchestrator.ts

export type WorkflowMode =
  | 'interactive'  // 关键节点暂停等待用户确认（默认）
  | 'delegated';   // 全自动推进，仅高风险时中断

export interface MultiAgentOrchestrator {
  createDAG(goal: string, roles: AgentRole[]): Promise<AgentDAG>;
  executeDAG(dagId: string, mode: WorkflowMode): Promise<void>;
  pauseDAG(dagId: string): Promise<void>;
  cancelDAG(dagId: string): Promise<void>;
  getDAGStatus(dagId: string): Promise<AgentDAG>;
  subscribeToEvents(dagId: string, handler: DAGEventHandler): () => void;
}

// DAGEvent 新增两个事件
export type DAGEvent =
  | { type: 'node_started'; nodeId: string; timestamp: number }
  | { type: 'node_completed'; nodeId: string; output: unknown; timestamp: number }
  | { type: 'node_failed'; nodeId: string; error: string; timestamp: number }
  | { type: 'edge_activated'; edgeId: string; timestamp: number }
  | { type: 'dag_completed'; result: unknown; timestamp: number }
  // 新增：interactive 模式下，关键节点需用户确认
  | { type: 'human_approval_required'; nodeId: string; plan: string; options: string[] }
  // 新增：delegated 模式下遇到高风险操作，强制降级为交互
  | { type: 'risk_escalation'; nodeId: string; riskDetail: string };

/**
 * 执行模式行为对照：
 *
 * interactive（默认）:
 *   - 方案选择、子 Agent 分配完成后触发 human_approval_required
 *   - 用户确认后继续；取消则终止当前节点
 *
 * delegated:
 *   - 阶段间自动推进，自动选推荐方案
 *   - 仅 risk_escalation 时中断，强制等待用户决策
 *   - 用户响应后可选择继续（恢复 delegated）或切换为 interactive
 */
```

##### 3. 子 Agent 身份协议（补入 `AgentRole`）

子 Agent 接收 prompt 时需识别自身身份，跳过路由逻辑直接执行任务：

```typescript
// packages/multi-agent/src/roles.ts

export interface AgentRole {
  id: string;
  displayName: string;
  description: string;
  systemPrompt: string;
  allowedTools: string[];
  model?: string;
  maxTokens?: number;

  // 新增：子 Agent 身份标记
  /**
   * 注入到子 Agent prompt 头部的身份前缀。
   * 子 Agent 收到含此前缀的 prompt 时，跳过完整路由（无需评分/追问/确认），
   * 直接执行任务，输出纯结果（不包装状态格式）。
   *
   * 默认值：'[SUB-AGENT] Execute directly. Skip routing. Return results only.'
   * 可按团队约定自定义，但必须在 orchestrator 与子 Agent 侧保持一致。
   */
  subAgentPromptPrefix?: string;
}

// Orchestrator 在派发子任务时自动注入前缀
function buildSubAgentPrompt(role: AgentRole, task: string): string {
  const prefix = role.subAgentPromptPrefix
    ?? '[SUB-AGENT] Execute directly. Skip routing. Return results only.';
  return `${prefix}

${task}`;
}
```

**作用**：入口 Agent 走完整路由协议（评估→确认→执行），子 Agent 跳过路由直接执行，避免子 Agent 重复追问用户、产生冗余交互。

##### 4. 失败处理与升级策略（补入 `DAGNode`）

```typescript
// packages/multi-agent/src/dag.ts

export interface RetryPolicy {
  maxRetries: number;       // 最大重试次数，默认 1
  backoffMs: number;        // 指数退避基数（ms），默认 1000
  escalateOnExhaustion: boolean; // 重试耗尽后是否上报主 Agent 做根因分析，默认 true
}

export interface DAGNode {
  id: string;
  type: 'orchestrator' | 'subagent' | 'tool' | 'human_input';
  agentRole?: AgentRole;
  label: string;
  status: 'pending' | 'running' | 'completed' | 'failed' | 'skipped';
  input?: unknown;
  output?: unknown;
  startedAt?: number;
  completedAt?: number;
  durationMs?: number;

  // 新增：失败处理策略
  retryPolicy?: RetryPolicy;
  /**
   * 失败升级处理流程（当 retryPolicy.escalateOnExhaustion = true 时）：
   *
   * 1. 收集失败信息：错误信息、已尝试次数、节点输入/输出
   * 2. 主 Agent 执行根因分析（五维）：
   *    a. 根因分类（逻辑错误/依赖缺失/环境问题/输入格式/模型能力）
   *    b. 为何之前的重试未生效
   *    c. 是否有其他节点存在同类问题（系统扫描）
   *    d. 修复建议（可操作）
   *    e. 是否需要用户介入
   * 3. 若可自动修复 → 修改任务描述后重新派发
   * 4. 若需用户介入 → 触发 human_approval_required 事件，展示根因分析结果
   */
  failureEscalationLog?: FailureEscalationRecord[];
}

export interface FailureEscalationRecord {
  attempt: number;
  error: string;
  rootCauseAnalysis?: {
    category: 'logic_error' | 'missing_dependency' | 'env_issue' | 'input_format' | 'model_capability';
    whyRetryFailed: string;
    affectedNodes: string[];     // 同类问题的其他节点 ID
    fixSuggestion: string;
    requiresHuman: boolean;
};
  resolvedAt?: number;
}
```

### 模块 W：浏览器自动化（Browser Automation）

**设计目标**：Agent 可控制本地或远程 Chrome/Chromium 执行网页操作，结果（截图/DOM/文本）返回对话流。

**参考**：browser-use/browser-use（81K stars）、browser-use/desktop。

#### 浏览器工具接口

```typescript
export interface BrowserAutomationTool {
  // 导航
  navigate(url: string): Promise<PageSnapshot>;
  goBack(): Promise<PageSnapshot>;
  goForward(): Promise<PageSnapshot>;

  // 交互
  click(selector: string): Promise<PageSnapshot>;
  type(selector: string, text: string): Promise<PageSnapshot>;
  scroll(direction: 'up' | 'down', amount: number): Promise<void>;
  selectOption(selector: string, value: string): Promise<void>;

  // 信息提取
  screenshot(): Promise<Uint8Array>;
  getDOM(): Promise<string>; // 简化 DOM 文本
  extractText(selector?: string): Promise<string>;
  getCurrentUrl(): Promise<string>;

  // 标签管理
  openTab(url?: string): Promise<string>; // 返回 tabId
  closeTab(tabId: string): Promise<void>;
  switchTab(tabId: string): Promise<void>;

  // 会话
  close(): Promise<void>;
}

export interface PageSnapshot {
  url: string;
  title: string;
  screenshot?: string; // base64
  domSummary?: string; // 关键 DOM 摘要
}
```

#### 反馈循环（Agent ↔ Browser）

```
Agent 发出 browser_action tool_use
    ↓
BrowserAutomationTool 执行
    ↓
返回 PageSnapshot（截图 + DOM 摘要）
    ↓
截图作为 image content 注入下一轮 LLM 上下文
    ↓
Agent 根据视觉反馈决定下一步操作
```

#### 平台差异

| 能力             | 桌面端（Tauri）   | 移动端                 |
| ---------------- | ----------------- | ---------------------- |
| 本地 Chrome 控制 | Playwright（CDP） | 不支持                 |
| 远程 CDP         | 支持              | 支持（连接桌面端代理） |
| In-App WebView   | Tauri WebView     | Expo WebView（受限）   |
| 截图反馈         | 完整支持          | 仅远程代理模式         |

#### 权限模型

- 每次 browser_action 触发权限提示（参考模块 O）
- 用户可配置「信任域名白名单」，白名单内操作无需逐次确认
- 敏感操作（表单提交、文件下载）始终需要 `once` 确认

**包位置**：`packages/browser-automation/`

### 模块 Y：工作流可视化编排（低代码拖拽）

**UI 组件**（基于 React Flow）：

**参考**：Dify（工作流编排）、FastGPT（可视化流程）。

### 模块 Z2：CI/CD 代码库深度集成（GitHub 事件触发）

**设计目标**：LLM 可在对话流中动态渲染交互组件（表单/表格/图表/审批按钮），用户操作结果回传 Agent 继续处理。

**参考**：CopilotKit AG-UI 协议（29K stars）。

```typescript
export type GenerativeComponentType =
  | 'form' // 动态表单（JSON Schema 驱动）
  | 'table' // 数据表格（含排序/过滤）
  | 'chart' // 图表（bar/line/pie）
  | 'approval' // 审批按钮（approve/reject + 备注）
  | 'code_diff' // 代码 diff 预览（含接受/拒绝）
  | 'file_list'; // 文件列表（含下载/预览）

export interface GenerativeUIMessage {
  type: 'generative_ui';
  component: GenerativeComponentType;
  props: Record<string, unknown>; // 严格 schema 校验，防 XSS
  onSubmit?: string; // 白名单回调 ID，不允许任意代码
}

export interface GenerativeUIRenderer {
  render(message: GenerativeUIMessage): React.ReactNode;
  handleSubmit(callbackId: string, data: unknown): Promise<void>;
}
```

**安全模型**：

- `props` 通过 Zod schema 严格校验，拒绝任意 HTML/JS 注入
- `onSubmit` 只允许白名单 callback ID，不执行任意代码
- 组件类型白名单（`GenerativeComponentType`），拒绝自定义组件

**UI 渲染流**：

```
LLM 输出 GenerativeUIMessage（JSON）
    ↓
GenerativeUIRenderer 解析 + 校验 props
    ↓
挂载对应 React 组件（form/table/chart 等）
    ↓
用户操作 → onSubmit → Agent 继续对话
```

**包位置**：`packages/generative-ui/`
