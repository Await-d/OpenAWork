# 260321 UI 数据联通补充方案

## 任务概述

当前 UI 组件已全部接入（shared-ui 67 个组件），但存在「组件渲染但数据为空/硬编码」的问题。
本方案专项覆盖 UI 与真实数据的打通，补全所有 UI 空状态/空数组/硬编码问题。

**关键背景**：
- `@openAwork/shared` 的 `StreamChunk` 协议定义了 `text_delta`/`tool_call_delta`/`done`/`error` 四种事件
- ChatPage 的 stream 客户端（`useGatewayClient`）只消费了 `text_delta`，其余类型完全未处理
- `PlanPanel`/`AgentVizPanel`/`AgentDAGGraph` 组件已渲染，但数据永远是空数组

## Complexity Assessment

- 原子步骤：8+ → +2
- 并行流：ChatPage stream解析 / SettingsPage持久化 / Gateway端点 / FileSearch 四条独立流 → +2
- 涉及模块：web + shared + web-client + agent-gateway → +1
- 单步 >5min → +1
- OpenCode 可用 → -1
- **Total: +5 → Full orchestration**
- **Routing rationale**: stream 事件解析跨 shared/web-client/ChatPage 三层，gateway 端点补全独立，可并行推进。

## 现状分析

### UI-01：ChatPage 右侧面板空数据

**根因**：`GatewayWebSocketClient.onmessage` 分发所有 `StreamChunk`，但 ChatPage 中 `client.stream()` 的回调只处理了 `text_delta`：
```typescript
client.stream(sid, text, {
  onDelta: (delta: string) => { ... },   // 只有文字
  onDone: () => { ... },
  onError: (code: string) => { ... },
});
```
`tool_call_delta`（含 `toolName`/`inputDelta`）从未被解析进 `planTasks`/`agentEvents`。

**修复方向**：扩展 `useGatewayClient` 的 stream 接口，新增 `onToolCall` 回调，在 ChatPage 中累积 tool call 事件更新 `planTasks` 和 `agentEvents`。

### UI-02：ContextPanel token 硬编码
```tsx
// Layout.tsx:1089
totalTokens={workspace.workingDirectory ? 500 : 0}
```
**修复方向**：从 stream `done` 事件的 `usage` 字段（待 Gateway 补充）或工作区文件树大小粗估（每文件 ~4 chars/token）计算。

### UI-03：FileSearch 空函数
```tsx
// Layout.tsx:1042
onSearch={async () => [] as FileSearchResult[]}
```
**修复方向**：
1. Gateway 新增 `GET /workspace/search?q=&path=` 端点（grep 文件内容）
2. `useWorkspace` hook 新增 `searchFiles(q, path)` 方法
3. Layout 传入真实回调

### UI-04：BudgetAlert 硬编码
```tsx
// SettingsPage.tsx:476
<BudgetAlert currentCostUsd={5.33} budgetUsd={10} onDismiss={() => {}} />
```
**修复方向**：复用 SettingsPage 已有的 `usageRecords`/`monthlyCostUsd` state（已从 API 获取），传入真实值。

### UI-05：Provider 配置持久化
`DEFAULT_PROVIDERS` 硬编码在前端，SettingsPage 的 `providers` state 只存 localStorage。
**修复方向**：Gateway 新增 `GET/PUT /settings/providers` 端点，SettingsPage 初始化时从 Gateway 拉取，修改时同步写入。

### UI-06：Gateway 骨架端点（同 FIX-01~03）
`/settings/mcp-status`、`/settings/diagnostics`、`/settings/workers` 均返回硬编码空数组。
**修复方向**：同优先级调整方案 FIX-01~03。
