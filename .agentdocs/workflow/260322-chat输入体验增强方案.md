# 260322 Chat 输入体验增强方案

## Task Overview

用户要求补强聊天输入法/输入框相关能力，范围包括：

1. 会话快捷指令操作增强
2. 输入框内置指令弹出与使用增强（`/`、`@`）
3. 粘贴图片支持

目标是在当前 Web Chat 输入区上提供更强的命令发现与快捷输入体验，而不是单纯保留全局 Command Palette。

## Current Analysis

- `apps/web/src/pages/ChatPage.tsx` 当前只有基础 textarea、Enter 发送、文件选择附件与本地图片预览。
- `packages/shared-ui/src/CommandPalette.tsx` 已有通用命令弹层，但它是全局模态，不适合输入框内的 slash/@ 场景。
- `ChatPage` 已经有 `attachedFiles` / `attachmentItems` / `ImagePreview` 路径，可复用来承接粘贴图片。
- 当前没有 `onPaste`、没有 slash command popup、没有 mention popup，也没有会话快捷插入项。

## Complexity Assessment

- Atomic steps: 7+ → +2
- Parallel streams: composer popup / session quick actions / image paste → +2
- Modules/systems/services: web chat + shared-ui attachment UI + docs → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 这是输入体验的多点位改造，需要统一在一份方案中跟踪 slash、mention、paste 三条能力的 red→green 证据与回归验证。

## Acceptance Criteria

- 在输入框输入 `/` 时出现可点击/键盘选择的命令列表
- 在输入框输入 `@` 时出现当前工作区文件相关的选择/引用列表
- 粘贴图片后进入附件栏，并显示图片预览
- 不破坏现有发送、附件删除、回车发送行为
