# Team 阶段2写入路径实施

## Task Overview

在 Stage 1 已完成 TeamWorkspace 根边界的基础上，继续推进 **Stage 2：写入路径切换**。本阶段目标是把 Team 内创建线程 / 任务 / 模板 / 消息的入口逐步从普通 `/sessions.create` 与 shared-session 包装层中剥离出来，建立真正的 Team-owned write path。

## Current Analysis

- Stage 1 已经完成：
  - TeamWorkspace 根接口已落地
  - web-client TeamWorkspace contract 已落地
  - `/team/:teamWorkspaceId` 已作为最小 workspace-first 根锚点接入
- 当前仍未完成的部分：
  - Team 新建线程仍未进入 Team-owned 对象链
  - Team 模板 / 任务 / 消息的写路径仍存在 user-scoped 平面写入或旧桥接语义
  - Team 页面部分创建入口仍停留在 reference data glue 层的兼容写法上

## Solution Design

- Stage 2 只做写路径，不提前推进完整读模型切换。
- 最小实施顺序：
  1. 先确定最小 Team-owned write API 切口
  2. 再补 web-client write contract
  3. 最后切前端创建入口，避免先改 UI 再猜后端语义
- 本阶段不碰：
  - 办公室画布深化
  - interaction-agent 主流程
  - review/artifact 完整对象化

## Complexity Assessment

- Atomic steps: 5+（gateway / web-client / web 创建入口 / 验证 / git） → +2
- Parallel streams: yes（gateway/client/frontend 切口分析） → +2
- Modules/systems/services: 3+（gateway / web-client / web） → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 写入路径切换涉及多层 contract 与入口重连，且必须避免把 Stage 3/4 混进来，适合用独立工作流执行。
