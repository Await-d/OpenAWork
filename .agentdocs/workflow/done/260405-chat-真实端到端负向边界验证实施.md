# 260405 Chat 真实端到端负向边界验证实施

## Task Overview

在真实 gateway 驱动的刷新恢复、停止、权限批准恢复、问题回答恢复 E2E 已打通后，继续补 `permission reject` 与 `question dismiss` 两条负向真实边界验证。

## Current Analysis

- 现有 live E2E 已覆盖正向恢复链。
- 还缺两个重要负向收口：
  - 权限请求被拒绝后，页面如何稳定退出等待态/运行态
  - 提问请求被暂不回答/关闭后，页面如何稳定退出等待态

## Solution Design

- 复用现有 live upstream / gateway / web 多进程配置；
- 复用已打通的 permission / question 真实触发链；
- 新增两条真实 E2E：
  - refresh → permission_asked → reject
  - refresh → question_asked → dismiss

## Notes

- 本次不新增协议功能，复用现有真实 live 验收基础设施。
- mock upstream 新增 prompt 分支：
  - `真实 gateway 权限拒绝验证` → 命中 bash tool call
  - `真实 gateway 提问关闭验证` → 命中 question tool call
- `question dismiss` 的真实终态不应再强绑问题文本本身消失，因为问题卡片/历史文本可能仍作为 transcript 保留。更稳定的负向断言应是：
  - 等待回答 prompt 消失
  - `提交回答` 操作入口消失
  - body 不再出现 `会话等待处理`
  - transcript 出现 `已忽略，等待进一步处理。`
- 通过验证：
  - `pnpm --filter @openAwork/web run test:e2e:attach-live`
  - `pnpm --filter @openAwork/web typecheck`
  - `pnpm --filter @openAwork/web build`
- Oracle 最终复核结论：`permission reject` 断言已足够可靠；`question dismiss` 在补上 idle 外显断言后也达到可接受的验证强度，未发现新的关键缺陷。
