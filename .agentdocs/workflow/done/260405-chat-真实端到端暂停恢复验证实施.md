# 260405 Chat 真实端到端暂停恢复验证实施

## Task Overview

在真实 gateway 驱动的刷新恢复与刷新后停止 E2E 已打通后，继续补真实的 `question_asked / paused / refreshed / answered / resumed / completed` 浏览器端到端验证。

## Current Analysis

- 当前真实 E2E 已覆盖刷新恢复到最终消息、刷新后 stop 控制恢复。
- 仍缺一个关键运行态边界：流在提问工具处暂停后，刷新页面是否还能保留等待回答的 paused 状态，并在真实回答提交后恢复执行直到完成。

## Solution Design

- 复用现有 live upstream / gateway / web 多进程配置；
- 通过 mock upstream 生成真实 `question` tool call，触发 gateway 提问暂停；
- 在浏览器中验证：
  - `question_asked` 产生
  - paused 状态与待回答 prompt 可见
  - 刷新后 paused 状态仍可见
  - 真实回答后 resumed 并最终完成
