# .agentdocs/workflow/260326-子代办双lane实现.md

## 任务概览

基于已完成的《260326-子代办双lane方案》，按 TDD + BDD 分阶段实现会话级双 lane todo（`main` + `temp`）。要求每完成一个功能切片，都必须先通过对应测试与手工验证，再进入下一步，并按仓库现有提交规范创建独立 git commit。

## 当前分析

- 已有详细设计冻结：v1 为固定双 lane；旧 `todowrite/todoread` 保持 main-only；新增 `subtodowrite/subtodoread` 与 `/sessions/:id/todo-lanes`；Web/Desktop 先接，Mobile 后置。
- 代码层现状：`services/agent-gateway/src/todo-tools.ts` 负责 todo 核心 CRUD；`services/agent-gateway/src/routes/sessions.ts` 提供 session detail 与 `/todos`；`apps/web/src/pages/ChatPage.tsx` 与 `right-panel-sections.tsx` 负责主视图与右侧展示。
- 用户要求按功能切片自动推进，并在每个切片通过后单独提交，因此实施必须拆成可回滚的原子阶段。

## Complexity Assessment

- Atomic steps: 5+ → +2
- Parallel streams: 方案已冻结，但实现涉及 gateway / web / tests / git 提交流程 → +2
- Modules/systems/services: services + packages + apps/web + apps/desktop 复用 → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 这是一个多阶段、多模块、带强验证与分批提交约束的实现任务，需要持续同步工作流与阶段状态。

## Acceptance Criteria

- Phase 1 完成后：数据库与 gateway CRUD 支持 `main/temp` 双 lane，旧 main 行为保持兼容；QA：gateway 相关测试通过 + 路由/函数手工读取输出正确。
- Phase 2 完成后：`subtodowrite/subtodoread` 与 `/sessions/:id/todo-lanes` 可端到端读写 temp；QA：工具/路由测试通过 + 手工调用返回符合预期。
- Phase 3 完成后：Web/Desktop 可分别展示 main 与 temp，主聊天区仍仅显示 main；QA：前端测试通过 + 本地渲染行为验证。
- 每个 phase 完成后：修改文件 LSP 无新错误、阶段测试通过、手工 QA 有实际输出、git 提交成功且提交信息不含用户禁止的 trailer。
