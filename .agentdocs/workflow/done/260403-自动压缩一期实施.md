# 自动压缩一期实施

## Task Overview

为 OpenAWork 的主模型输入链补齐自动压缩第一阶段，目标是把当前“固定消息窗口 + 手动 `/compact` 卡片”升级为“自动触发 + compact boundary + 结构化 summary + UI artifact 不污染模型上下文”的可验证闭环。

## Current Analysis

- 当前主模型输入链位于 `services/agent-gateway/src/routes/stream-model-round.ts -> buildUpstreamConversation()`。
- 当前 `buildUpstreamConversation()` 主要做固定窗口裁切（默认 12 条）与 tool call / result 配对保护，但对被裁掉的历史没有结构化保留。
- 当前 `/compact` 实现在 `services/agent-gateway/src/routes/commands.ts`，更像“生成一张 compaction 卡片 + metadata”，而不是模型侧真实压缩。
- `command-card:*` 这类消息目前会以普通 assistant 文本写回会话，存在再次进入模型上下文的风险。

## Solution Design

- 保持 `buildUpstreamConversation()` 兼容现有调用方；新增仅供主模型轮次使用的“prepared conversation”构建函数。
- 在 prepared conversation 中，当固定窗口将丢弃更早消息时，自动合成：
  - compact boundary system message
  - 结构化 summary system message
- 为自动压缩补齐 run event 与 session metadata 留痕，但不把 UI 卡片 JSON 重新喂回模型。
- `/compact` 命令沿用现有 UI 交互表面，但摘要改为结构化文本，并把 command-card 从 upstream conversation 中排除。

## Complexity Assessment

- Atomic steps: 5+ → +2
- Parallel streams: no → 0
- Modules/systems/services: 3 (`agent-gateway` 路由、message store、agentdocs/workflow) → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 4
- **Chosen mode**: Full orchestration
- **Routing rationale**: 这是一次跨 gateway 主链与持久化行为的改造，既要留下 workflow 决策痕迹，也要分阶段验证自动压缩、上下文构造与命令卡片隔离三条子链，因此采用 full orchestration 但在 OpenCode 中由当前主会话直接执行。
