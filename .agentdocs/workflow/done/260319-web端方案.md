# .agentdocs/workflow/260319-web端方案.md

## 任务概览

补充当前方案中缺失的 **Web 端**支持：用户通过浏览器直接访问，无需安装任何客户端。

本方案与现有方案的关系：

- **复用**：主计划 `Agent Gateway`（T-10~T-14）作为统一后端，无需重复开发 AI 逻辑
- **新增**：`apps/web` — React SPA Web 客户端
- **新增**：`packages/shared-ui` — 跨端共享 React 组件库
- **新增**：`packages/web-client` — Web 端 Gateway 连接层

本方案与前六份方案**完全隔离**，独立时间线，独立交付。

---

## 架构总览

```
┌─────────────────────────────────────────┐
│  Agent Gateway（Fastify + WS/SSE）       │
│  agent-core · Tool Registry · SQLite    │
└────────────────┬────────────────────────┘
                 │ HTTP / WebSocket
       ┌─────────┼──────────┐
       │         │          │
   app/mobile  app/desktop  apps/web ← 新
   (Expo)      (Tauri)      (React SPA)
```

**核心原则**：Gateway 是唯一 AI 执行中枢，所有端通过相同 HTTP API + WebSocket 接入，AI 逻辑零重复。

---

## 各端定位对比

| 维度 | Web 端 | 移动端 | 桌面端 |
|------|--------|--------|--------|
| 安装要求 | 无（浏览器访问） | 需安装 App | 需安装软件 |
| 适用场景 | 临时访问、团队共享、服务器部署 | 移动办公 | 本地文件操作 |
| 文件系统 | File API 上传/下载 | 沙箱内 | 完整 Tauri 权限 |
| MCP transport | SSE / WebSocket | SSE / WebSocket | + stdio |
| 离线能力 | PWA UI Shell 缓存 | 完整支持 | 完整支持 |

## 实时通信接口

```typescript
export class GatewayWebSocketClient {
  connect(gatewayUrl: string, token: string): void;
  send(message: ClientMessage): void;
  onStream(cb: (chunk: StreamChunk) => void): () => void;
  onToolCall(cb: (event: ToolCallEvent) => void): () => void;
  onPlanUpdate(cb: (plan: TaskPlan) => void): () => void;
  disconnect(): void;
  // SSE 降级
  connectSSE(gatewayUrl: string, token: string): void;
}
```

## 部署模式

```bash
# 模式 1：静态托管（推荐）
npm run build --workspace=apps/web
# dist/ → Vercel / Netlify / Nginx

# 模式 2：Gateway 内嵌静态 serve（自托管单进程）
fastify.register(fastifyStatic, { root: path.join(__dirname, 'web-dist') })

# 模式 3：Docker 一键部署
docker-compose up  # Gateway + Web UI

# 模式 4：PWA 安装到桌面（用户浏览器操作）
```

## 风险矩阵

| 风险 | 概率 | 影响 | 控制措施 |
|------|------|------|----------|
| WebSocket 被企业代理拦截 | 中 | 高 | SSE 自动降级；文档说明 HTTPS+WSS 要求 |
| JWT XSS 风险 | 中 | 高 | httpOnly cookie；严格 CSP；禁止 localStorage |
| Web 文件系统访问受限 | 高 | 中 | 上传/下载模式；Artifacts URL 分发 |
| shared-ui 跨端样式差异 | 中 | 中 | Storybook 文档化；Playwright 视觉回归 |
| Docker 镜像体积 | 低 | 低 | Alpine 基础镜像；multi-stage build |
