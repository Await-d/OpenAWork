# Agent 管理页面方案

## Task Overview

为 Web 端新增一个可查看全部 Agent 的页面，并提供最小但真实可用的管理/编辑能力，方便用户统一查阅当前可用 Agent、了解其角色与别名，并按用户维度维护管理配置。

## Current Analysis

- `apps/web/src/App.tsx` 当前已有独立页面路由体系，`Skills / Team / Usage / Schedules` 等都以一级页面存在，新增 `AgentsPage` 符合现有结构。
- `apps/web/src/components/layout/RailIcon.tsx` 和 `NavRail.tsx` 决定侧栏主导航；新增 Agent 页面应与 `Skills` 同级，而不是继续膨胀 `SettingsPage`。
- `services/agent-gateway/src/routes/capabilities.ts` 已提供只读 `capabilities` 列表，并且 Agent 已携带 `canonicalRole` 与 `aliases`，适合作为页面的主事实源。
- 当前未发现现成的 Agent 编辑接口；现有用户级配置存储模式是 `user_settings` 表，Settings/Channels/MCP 都以此承载用户偏好。
- 因此最小真实管理能力不应尝试修改内置 reference agent 本体，而应新增用户级 `agent_preferences`（或同等 user_settings key）来承载诸如隐藏/收藏/备注/自定义显示名等管理编辑信息，再与 capabilities 列表聚合展示。

## Solution Design

### 目标范围

本次页面必须满足三件事：

1. **查看全部 Agent**：展示当前 capabilities 中所有 `kind === 'agent'` 的条目。
2. **便于查阅**：支持搜索、按 core role/source 过滤、查看 canonical role / aliases / 描述。
3. **可管理编辑**：提供用户级配置编辑，而非修改 reference agent 定义本身；初始支持 `displayNameOverride`、`note`、`favorite`、`hidden` 四类最小管理字段。

### 架构选择

- **读取事实源**：`/capabilities`
- **用户管理配置**：新增 gateway 路由（建议 `/agents/preferences`）读写 `user_settings`
- **前端聚合模型**：页面把 capabilities agent 列表与 preferences 按 `agentId` 合并，形成可渲染视图模型
- **UI 形态**：独立 `AgentsPage`，左侧列表 / 顶部筛选 / 右侧详情编辑面板或页内编辑卡片

### 数据模型草案

```ts
interface AgentPreferenceRecord {
  agentId: string;
  displayNameOverride?: string;
  note?: string;
  favorite: boolean;
  hidden: boolean;
  updatedAt: string;
}
```

```ts
interface ManagedAgentViewModel {
  id: string;
  label: string;

  source: string;
  canonicalRole?: CanonicalRoleDescriptor;
  aliases?: string[];

  note?: string;
  favorite: boolean;
  hidden: boolean;
}
```

- 本次“管理编辑”按最简单有效解释实现为**用户级 Agent 管理偏好**，不直接修改 reference agent 定义或 capabilities 基础清单。
- 若后续需要真正的自定义 Agent 创建/删除，应作为第二阶段能力，不与本次页面目标耦合。
- 已完成的自动化验证：`agents-routes.test.ts`、`AgentsPage.test.tsx`、`App.test.tsx`、`ChatPage.commands.test.tsx`、`capabilities.test.ts`。
- 已完成的手工验证：`AgentsPage` 渲染脚本确认页面标题/自定义显示名/canonical role/保存动作均出现；`PUT/GET /agents/preferences` 真实往返确认保存与回显闭环。
