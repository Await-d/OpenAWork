# .agentdocs/workflow/260330-后台保险操作对齐复检.md

## 任务概览

对当前仓库再次执行“文件变更记录、数据库对话记录、文件落地日志、会话日志、全局权限、工作区权限等后台保险操作”与 `temp/opencode` / `temp/oh-my-openagent` 的逐项对齐复检，并继续迁移缺失值与关键安全机制。

## 当前分析

- 前几轮已把工具面、子代理模型与一批参考能力迁移进当前仓库，但后台保险链路仍可能存在“功能能跑、审计与持久化字段不全”的差异。
- 用户这次点名的范围集中在 durable state：会话主事实、工具审计、文件变更证据、权限范围与持久化、工作区级安全状态，以及与这些数据关联的日志链路。
- 这类缺口的风险不在 UI，而在事后追溯、恢复、权限边界与跨会话安全保证，因此必须按数据字段和持久化点逐项对照。

- Modules/systems/services: agent-gateway + agent-core + shared + temp references + .agentdocs → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1

- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 本轮是后台持久化与权限保险链路的深度对比，不只是工具名核对；需要保留差异清单、迁移记录与验证证据，避免遗漏隐蔽但高风险的数据字段差异。
