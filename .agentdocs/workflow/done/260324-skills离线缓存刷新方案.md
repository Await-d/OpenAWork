# .agentdocs/workflow/260324-skills离线缓存刷新方案.md

## 任务概览

将 Skills 页面第三方注册源改造为离线优先：添加后立即同步第三方源技能列表到本地缓存，页面默认从本地缓存读取，手动刷新时再拉取远端最新数据并替换本地缓存。

## 当前分析

- `apps/web/src/pages/SkillsPage.tsx` 首屏直接调用 `/skills/search`
- `services/agent-gateway/src/routes/skills.ts` 当前会并发聚合官方源、用户第三方源、GitHub 内置源与 builtin 数据
- `packages/skill-registry/src/client.ts` 对所有启用源发起远端搜索，缺少本地缓存与超时/离线优先策略
- `services/agent-gateway/src/db.ts` 中 `registry_sources` 仅存源配置，不存同步结果

## 方案设计

- 在 gateway SQLite 中新增“源缓存快照”存储，按 `user_id + source_id + skill_id` 保存第三方源技能元数据
- `/skills/search` 默认返回本地缓存 + builtin/GitHub 数据，避免首屏依赖第三方远端请求
- 新增显式刷新模式，由页面刷新按钮触发远端同步；同步成功后用最新结果覆盖本地缓存
- 添加/启用第三方源时立即执行一次同步，确保后续离线可读
- 保持安装链路不变，本次聚焦页面浏览性能与离线列表访问
