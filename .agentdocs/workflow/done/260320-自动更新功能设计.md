# 自动更新功能设计方案

## 任务概览

为 OpenAWork 跨平台客户端（Tauri v2 桌面端 + Expo 移动端）设计并实现完整的自动更新系统，覆盖：

- Desktop：tauri-plugin-updater 完整接入（Rust + 配置 + 签名 + 通道）
- Mobile：expo-updates OTA 完整接入（配置 + CI + 通道策略）
- 灰度发布与自动回滚
- 更新通知 UI
- 崩溃后自动回滚保障

## 方案设计

### 核心架构决策

1. **桌面端通道模型**：`stable`（GitHub Releases 正式发布）、`preview`（预发布）两通道
2. **移动端通道模型**：`production`、`preview` EAS Update 通道，一一对应 EAS build channel
3. **通道隔离**：桌面端通过不同 updater endpoint URL 区分通道；移动端通过 EAS channel 隔离
4. **灰度策略**：5% → 25% → 100%，每步观察期 ≥ 24 小时（桌面端依赖 GitHub Release 可见性控制；移动端依赖 `eas channel:rollout`）
5. **自动回滚触发条件**：Sentry 错误率 > 5% 或崩溃率 > 2%，30 分钟观察窗口
6. **签名要求**：Tauri updater 必须使用 Ed25519 签名验证安装包完整性；移动端通过 EAS 平台签名保障

## 关键技术细节

### Desktop Tauri v2 更新配置（目标状态）

```toml
# Cargo.toml 新增
[dependencies]
tauri-plugin-updater = "2"
```

```rust
// lib.rs 新增
.plugin(tauri_plugin_updater::Builder::new().build())
```

```json
// tauri.conf.json — plugins 块新增
"plugins": {
  "updater": {
    "pubkey": "<Ed25519 公钥>",
    "endpoints": [
      "https://github.com/openAwork/openAwork/releases/latest/download/latest.json"
    ],
    "dialog": false
  },
  "shell": { "open": true }
}
```

```json
// capabilities/default.json 新增权限
"updater:default"
```

### Desktop 更新通道隔离策略

- **stable**：`/releases/latest/download/latest.json`（GitHub "latest" release，忽略 draft/prerelease）
- **preview**：`/releases/download/desktop-vX.Y.Z-preview/latest.json`（直接指向 prerelease tag）
- 通道选择在运行时通过 app 配置注入（构建时区分 stable/preview 的 tauri.conf.json 变体，或通过 Rust feature flag）

### release-desktop.yml 关键修复

```yaml
# 修复前：tag push 时 inputs.channel 为空，导致 releaseDraft: true
releaseDraft: ${{ inputs.channel != 'stable' }}

# 修复后：tag push 默认为 stable（正式发布不作 draft）
releaseDraft: ${{ inputs.channel == 'preview' }}
prerelease: ${{ inputs.channel == 'preview' }}
```

### Mobile expo-updates 配置（目标状态）

```json
// app.json 新增
"expo": {
  "updates": {
    "url": "https://u.expo.dev/<project-id>",
    "checkAutomatically": "ON_LOAD",
    "fallbackToCacheTimeout": 3000
  },
  "runtimeVersion": {
    "policy": "fingerprint"
  }
}
```

```ts
// useOtaUpdate hook 核心逻辑
import * as Updates from 'expo-updates';

export function useOtaUpdate() {
  const checkAndApply = async () => {
    const update = await Updates.checkForUpdateAsync();
    if (!update.isAvailable) return;
    await Updates.fetchUpdateAsync();
    await Updates.reloadAsync(); // 热重载，无需重新安装
  };
  // 启动时在后台静默检查，有更新时 toast 提示
  useEffect(() => { checkAndApply().catch(() => {}); }, []);
}
```

### 签名密钥管理

```bash
# 生成 Tauri updater Ed25519 密钥对（一次性操作）
pnpm tauri signer generate -w ~/.tauri/updater-key.pem
# 将私钥存入 GitHub secret: TAURI_SIGNING_PRIVATE_KEY
# 将公钥填入 tauri.conf.json plugins.updater.pubkey
```

## 风险与控制

| 风险 | 概率 | 影响 | 控制措施 |
|---|---|---|---|
| Desktop latest.json 不可访问（draft release） | 高 | 高 | 修复 release-desktop.yml tag push 默认通道逻辑 |
| Mobile OTA 与 native 版本不兼容（runtimeVersion 不匹配） | 中 | 高 | 使用 fingerprint 策略；不兼容时自动降级到 store 版本 |
| 自动更新打断用户工作（桌面端） | 中 | 中 | 静默下载，用户确认后才重启；提供「推迟」选项 |
| 签名私钥泄露 | 低 | 极高 | 仅存 CI secrets，本地不留副本；定期轮换策略文档化 |
| 灰度期间多版本共存导致 Gateway API 兼容问题 | 中 | 中 | Gateway 保持向后兼容 1 个大版本；version header 记录客户端版本 |

## 度量指标

- 更新推送到安装完成时间（P50/P95）

- OTA 覆盖率（Mobile：已获得 OTA 的活跃设备占比）
