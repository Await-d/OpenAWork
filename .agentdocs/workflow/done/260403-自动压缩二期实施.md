# 自动压缩二期实施

## Task Overview

在自动压缩一期的 runtime compaction 基础上，继续把 OpenAWork 推向 durable summary / session memory 方向：通过 session metadata 持久化累计 compaction memory，让主模型输入链能跨轮次复用已压缩历史，而不是每轮都从头重做旧历史摘要。

## Current Analysis

- 一期已经让 `buildPreparedUpstreamConversation()` 在固定安全窗口将丢弃历史时自动注入 compact boundary 与 structured summary。
- 当前 compaction 结果虽然会写入 `lastCompactionSummary`，但仍是“本轮结果”，缺少累计记忆与 covered boundary 概念。
- 仓内可复用的 durable condensed-context 载体主要是 `session metadata`、`session_snapshots.summary_json` 与 `/handoff` 的结构化交接摘要；没有现成的 Claude Code 风格独立 SessionMemory 子系统。

## Solution Design

- 二期不扩表，先采用 **metadata-backed session memory**：在 `metadata_json` 中持久化 `compactionMemory`，记录累计摘要字段、`coveredUntilMessageId`、累计压缩次数与最新边界信息。
- `buildPreparedUpstreamConversation()` 改为优先读取已持久化的 compaction memory，只对“尚未被 covered 的新遗漏消息”做增量摘要，并把新 delta 归并进累计 memory。
- 自动压缩与手动 `/compact` 统一走相同的 durable memory 归并逻辑，避免两套摘要语义分叉。
- 继续保持 UI artifact 与模型上下文隔离；二期的 durable memory 只作为 metadata 与 model-side summary 使用，不反向生成新的 command-card 噪音。
