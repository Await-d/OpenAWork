# .agentdocs/workflow/260402-usage持久化回归修复.md

## Task Overview
按既有 usage 方案设计修复 `services/agent-gateway/src/verification/verify-openai-responses.ts` 当前的 usage persistence 回归，恢复 `usage_records` 月度写入主链，并确认 chat / responses 路径的 usage 透传与验收脚本重新通过。

## Current Analysis
- 现有 AgentDocs 设计已经明确：usage 不能伪造，必须来自上游真实 usage；chat-completions 需显式 `stream_options.include_usage=true`，Responses 从 `response.completed/response.incomplete` 读取 usage。
- 当前失败现象：`verify-openai-responses.ts` 的 text 场景期望 `input_tokens=1`，实际读取为 0。
- Oracle 与 usage 链路追踪都指出：当前更可能的根因不是“没解析到 usage”，而是“`stream-model-round.ts` 把 usage 留在 `streamState.usage` 内部，没有在生产主链落库”；`persistMonthlyUsageRecord()` 仅在测试中出现，运行时没有接线。

## Solution Design
- 保持既有 usage 设计不回退，不放宽 verification 断言。
- 在 stream 公共执行主链补 usage 落库，而不是只在单一路由旁路修补。
- 优先检查并修复 `runModelRound()` / `handleStreamRequest()` 之间的 usage 透传与月度 upsert。
