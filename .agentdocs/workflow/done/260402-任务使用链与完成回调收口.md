# .agentdocs/workflow/260402-任务使用链与完成回调收口.md

## Task Overview
基于上一轮任务体系审计结果，继续把剩余 task 使用链与任务完成回调缺口按当前 fusion-native / reference way 收口，覆盖 `packages/web-client`、`apps/mobile` 与必要的 gateway 行为对齐，并恢复相应验证。

## Current Analysis
- 当前 `shared + agent-gateway + apps/web` 主链已能完成 task 状态收敛、父会话 `tool_result` 收敛与 `task_update` 事件消费。
- 剩余缺口集中在三类：
  1. `packages/web-client` 仍把流式消息收窄为 `StreamChunk`，会漏掉 `task_update/tool_result/permission/question` 等 `RunEvent`；
  2. `apps/mobile` 的 WS 鉴权方式仍停留在旧 body authorization，且未完整消费 `task_update`；
  3. `task-parent-auto-resume` 具备实现但 task 主路径是否应按参考方式启用，仍需结合既有文档与验收脚本裁定。

## Solution Design
- Phase 1 先收口 `packages/web-client` 的 `RunEvent` 类型与测试，避免通用客户端继续丢任务事件。
- Phase 2 再把 `apps/mobile` 对齐到当前参考方式：WS 握手 token、消息发送模式、`task_update` 事件消费，以及必要的任务状态对账。
- Phase 3 根据既有方案与脚本证据裁定并实现 `task-parent-auto-resume` 的参考行为，然后跑专项验证、类型检查与构建。

- Long step (>5 min): 是 → +1
- Persisted review artifacts: 是（需要方案与验收留痕） → +1
- OpenCode available: 是 → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
