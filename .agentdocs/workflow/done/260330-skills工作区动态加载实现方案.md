# .agentdocs/workflow/260330-skills工作区动态加载实现方案.md

## 任务概览

在现有 Skills 页面与 gateway 安装链路基础上，补齐“工作区本地 skills 动态发现、页面可安装/刷新、运行时可生效”的最小可用闭环。

目标不是重做整套 Skills 平台，而是在当前架构下增加本地工作区技能流：

- 扫描工作区内本地 skills
- 在页面中可见、可安装、可刷新
- 安装后纳入现有 `installed_skills` 与 `skill` / `skill_mcp` 生效链路
- 保持现有远程 registry / builtin / GitHub 市场行为不回退

## 当前分析

- Web 端已有 Skills 市场与已安装管理页面，当前只覆盖远程 registry 安装与启停。
- Gateway 已有 `/skills/install`、`/skills/search`、`/skills/installed` 等接口，但缺少“扫描工作区本地 skill 目录”的 HTTP 入口。
- `packages/skill-registry/src/installers/local.ts` 已有本地安装器与 watch 预留，但尚未接入 gateway。
- `packages/platform-adapter/src/agentskills-paths.ts` 已定义多平台 skills 目录候选路径，但未接入当前运行时。
- 当前 `installed_skills` 按 `(skill_id, user_id)` 存储，运行时 `skill` / `skill_mcp` 会在调用时读取已启用技能，因此只要本地 skill 能正确写入该表，就能复用现有运行时生效路径。

## 解决方案设计

- Gateway 新增“本地工作区 skills 扫描”读接口，基于 `WORKSPACE_ROOTS` 遍历候选目录并解析 `skill.yaml`。
- 本地扫描结果与远程市场列表分层展示：远程市场继续走 `/skills/search`，本地工作区技能走独立接口，避免污染离线缓存语义。
- Gateway 新增本地 skills 安装/刷新入口，直接调用 `SkillRegistryClientImpl.installFromLocal()` 或同等本地 manifest 安装路径，并把记录持久化进 `installed_skills`。
- 对本地技能保存 `source_id='local'` 或更细的本地来源标识，并在返回体中带上 `localPath` / `workspaceRelativePath` 供 UI 使用。
- Web Skills 页面补充“工作区本地技能”区域：支持加载列表、安装、刷新，并与现有已安装列表联动。
- 优先实现“手动刷新”而非真正文件监听热更新；若后端结构允许，再保留 watch 扩展点但不强依赖首版上线。
