# .agentdocs/workflow/done/260330-中文发布日志强制方案.md

## Task Overview

把“每次版本发布都需要中文更新日志总结”落成强制流程。目标是在现有自动版本/发布链上，让 `Prepare Release` 必填中文 summary，并把它持久化到仓库内、同时接入 desktop/mobile 的发布输出。

## Current Analysis

- 当前 `prepare-release.yml` 已能自动 bump、commit、push、打 tag / dispatch workflow，但用户要求发布稿不能存进本地仓库
- `release-desktop.yml` 会创建 GitHub Release，适合消费 workflow 运行时生成的中文发布稿，并在发布完成后补充安装包下载信息
- `release-mobile.yml` 没有 GitHub Release 产物，更适合消费同一份 runtime-only 发布稿并输出 workflow summary
- 发布稿需要通过 annotated tag / workflow input / 临时文件 artifact 在 GitHub Actions 内部传递

## Solution Design

- 新增发布日志脚本，在 workflow 运行时根据版本号、目标和中文 summary 生成发布稿正文
- `prepare-release.yml` 增加必填中文 `release_notes` 输入，并在 workflow 内生成临时 `release-notes.md`，不提交到仓库
- tag 驱动的 release workflow 通过 annotated tag 携带完整发布稿；mobile preview / 手动触发通过 `release_notes_body` input 透传发布稿正文
- `release-desktop.yml` 消费 runtime-only 发布稿作为 release body 基础正文，并在多平台安装包上传后自动追加下载链接；`release-mobile.yml` 读取同一份正文，使用首行中文摘要作为 OTA message，并把完整发布稿写入 workflow summary
