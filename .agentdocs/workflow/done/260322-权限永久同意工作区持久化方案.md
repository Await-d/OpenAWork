# 260322 权限永久同意工作区持久化方案

## Task Overview

用户要求重新定义“永久同意”的真实语义：

- 点击 **永久同意** 时，必须在当前工作区下生成配置文件保存授权
- 下次启动后，只要再次检查到该工作区，就不应重复询问权限

## Complexity Assessment

- Atomic steps: 3–4 → 0
- Parallel streams: no → 0
- Modules/systems/services: agent-core permission + workspace file → +1
- Long step (>5 min): no → 0
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 1
- **Chosen mode**: Lightweight
- **Routing rationale**: 虽然改动集中在 `PermissionManagerImpl`，但涉及文件持久化与重启读取语义，需要保留独立文档记录验证链路。

## Implementation Summary

- `packages/agent-core/src/permission/index.ts`
  - 新增 `workspaceRoot?: string`
  - `check(..., workspaceRoot?)` 支持在检查前加载工作区永久授权
  - `reply('permanent')` 会写入 `${workspaceRoot}/.openawork.permissions.json`
  - 新增 `loadWorkspacePermissions()` / `saveWorkspacePermissions()`
- `packages/agent-core/src/permission/index.test.ts`
  - 覆盖永久授权写入并在新 manager 实例中重载生效
  - 覆盖 `once / session` 不落盘

## Acceptance Criteria

- 选择“永久同意”时会在工作区根目录生成 `.openawork.permissions.json`
- 新 manager 在同一 `workspaceRoot` 下检查同一权限时返回 `permanent`
- `once / session` 不生成工作区文件
