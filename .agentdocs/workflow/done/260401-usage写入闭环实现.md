# .agentdocs/workflow/260401-usage写入闭环实现.md

## Task Overview

完成 usage 月度记录闭环的最终收口：在确认 protocol / round / stream / resume 已接线后，补齐 chat-completions 流式 usage 请求条件、真实集成验收与包级回归验证。

## Current Analysis

- 当前仓库在业务代码层面已经具备 usage 闭环核心能力：
  - `stream-protocol.ts` 已能解析 Responses 与 chat-completions 的 usage；
  - `stream-model-round.ts` 已把 round usage 向上返回；
  - `stream.ts` 与 `stream-runtime.ts` 已调用 `persistMonthlyUsageRecord()` 做月度累计写入。
- 剩余缺口并不在主链路代码，而在两件事：
  1. `chat_completions` 流式请求是否显式开启 usage 返回
  2. 真实验收是否证明 `usage_records` 会被写入
- 官方语义要求 OpenAI Chat Completions 流式请求显式启用 `stream_options.include_usage=true`，否则最后的 usage chunk 默认不会返回；Responses 则可在 `response.completed` / `response.incomplete` 中拿到 usage。

## Solution Design

1. 保持现有月度累计实现不回退，补齐 chat-completions 请求参数，使真实上游能返回 usage。
2. 在协议/请求单测中验证 `stream_options.include_usage` 已默认开启。
3. 在 `verify-openai-responses.ts` 中同时覆盖：
   - Responses 文本场景写入 `usage_records`
   - chat EOF 工具场景写入 `usage_records`
4. 以定向单测 + 集成验收 + 包级测试 + 构建作为最终验证门禁。
