# .agentdocs/workflow/260327-子代理任务默认权限收敛.md

## Task Overview

将当前子代理 task 权限“默认放开”的行为调整为更收敛的默认策略，确保默认值、序列化/反序列化、调用链与相关测试保持一致。

## Current Analysis

- 仓库近期已多次收敛 task、permission、child session 链路，这次修改需要优先遵循既有权限模型而不是局部硬改。
- 用户意图明确为“默认策略调整”，不是仅做文档说明，因此需要完成真实代码修改和验证。
- 已确认真正需要收紧的不是 `permission_requests` 审批表本身——gateway 的 `task` 工具本来就会审批；问题在于 task 创建出来的 child session 仍默认暴露 `task` 工具，导致子代理可继续递归委派。

## Solution Design

- 先定位子代理 task 权限默认值、fallback 和持久化边界。
- 仅修改最小必要文件，避免扩大到无关权限语义。
- 若已有测试覆盖默认值或配置 round-trip，则同步更新；若缺失，则补充回归测试。
