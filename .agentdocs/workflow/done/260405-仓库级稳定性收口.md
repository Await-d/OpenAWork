# 仓库级稳定性收口

## Task Overview
在 Buddy / Agent / 语音 / 默认回退链路完成后，继续把工作扩展到仓库级稳定性收口：修复 broader web/gateway 测试漂移、agent-core 原生依赖测试阻塞、mobile typecheck 环境缺口，并最终让根级 `pnpm typecheck` 与 `pnpm test` 全部通过。

## Current Analysis
- 初始状态下，Buddy 功能链已完成，但 broader suite 与根级验证仍未收口。
- `apps/web` 存在多处测试契约与时序脆弱点：`ChatPage.commands`、`Layout`、`TeamPage`、`FileEditorPanel`、`golden-transcript`、`ToolCallCard`、attach/recovery 相关断言。
- `services/agent-gateway` 存在多处典型夹具漂移：`settings-providers`、`session-run-events`、`session-todo-routes`、`stream-agent-resolution`，后续又收敛到 `verify-task-tool-failure-propagation` 与 `verify-claude-first-tools` 两条 verification 链。
- 根级还暴露出两类环境问题：`apps/mobile` 的 Expo/RN 类型解析、`packages/agent-core` 的 `better-sqlite3` native binding 缺失。

## Solution Design
- `web`：对齐当前实现更新测试契约，补齐 Team/App mock 覆盖面，收敛 `ChatPage` attach/recovery 用例到当前稳定行为，修复 `ToolCallCard` 非折叠分支缺失的恢复标签与状态文案。
- `gateway`：补齐缺失 mock 导出、修正过时断言、为 verification 放宽等待窗口，并修复 `ExitPlanMode` 通知 ID 冲突。
- `agent-core`：在 `sqlite-session-store.test.ts` 中使用可控内存 mock 替代对 `better-sqlite3` native binding 的硬依赖，保留 `SQLiteSessionStore` 行为契约测试。
- `mobile`：通过恢复工作区依赖环境，让 Expo/RN 类型与测试进入可验证状态。
