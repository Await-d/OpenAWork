# Buddy 语音参数与默认回退补全

## Task Overview
在现有 Buddy-Agent 绑定基础上，补齐两项建议能力：1) Agent 专属 Buddy 语音参数（`voiceVariant / voiceRate / voiceOutputMode`）绑定；2) 未绑定 Agent 的 Buddy 统一默认人格回退，不再按 `agentId` 派生不同默认 persona。

## Current Analysis
- 现有 binding 已支持外观与行为（`species / themeVariant / displayName / behaviorTone / injectionMode / verbosity`），但本轮前语音参数仍只存在于全局 companion preferences schema 中，前端 voice hook 未消费它们。
- 当前未绑定 Agent 时，Buddy 仍会按 `userEmail:agentId` 生成不同默认 persona，不符合“统一默认人格”目标。

## Solution Design
- 扩展 `CompanionAgentBinding`，增加 `voiceVariant / voiceRate / voiceOutputMode`。
- gateway helper 增加有效语音设置解析：绑定级 override 优先，全局 companion preferences 次之。
- `useBuddyVoicePreferences` 对外暴露 `effectiveVoice*`。
- `useBuddyVoiceOutput` 改为真实消费 `voiceVariant / voiceRate / voiceOutputMode`。
- 统一默认人格回退：若当前 Agent 无绑定，则一律回退到 `userEmail` 的默认 Buddy，而不是 `userEmail:agentId`。

## Complexity Assessment
- 原子步骤：5 个核心步骤（shared 类型 / gateway 解析 / voice hook / Settings UI / 验证） → +2
- 并行流：gateway helper 与 web UI/voice hook 可并行推进 → +2
- 模块/系统/服务：`packages/shared` + `services/agent-gateway` + `apps/web` → +1
- 单步 >5min：语音参数链路与回归验证超过 5 分钟 → +1
- 持久化审查产物：需同步 workflow / runtime / index → +1
- OpenCode 可用：→ -1
- **总分：6**
- **选定模式：Full orchestration**
- **路由理由：这是跨 shared 类型、gateway settings 真相源和 web voice/runtime 的真实能力补全，必须带验证证据收口。**
