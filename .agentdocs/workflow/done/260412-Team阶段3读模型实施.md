# Team 阶段3读模型实施

## Task Overview

在 Stage 1 已完成 TeamWorkspace 根边界、Stage 2 已完成最小 Team-owned thread create 的基础上，继续推进 **Stage 3：读模型切换**。本阶段目标是把 Team 页面主读链从 `sharedSessions / selectedSharedSessionId / runtimeTaskGroups` 驱动的旧投影，切换到 `TeamWorkspaceSnapshot + Team 子域列表` 驱动的 workspace-first 读模型。

## Current Analysis

- Stage 1 已完成：
  - TeamWorkspace 根接口与根 contract 已落地
  - `/team/:teamWorkspaceId` 已作为 workspace-first 路由锚点接入
- Stage 2 已完成：
  - Team UI 的“新建会话”已改走 Team-owned thread create
- 当前仍未完成的部分：
  - `useTeamCollaboration()` 仍以 `sharedSessions` / `selectedSharedSessionId` 为 Team 主状态来源
  - `team-runtime-reference-data.tsx` 仍把 raw shared/runtime projection 直接喂给页面
  - Team 主视图尚未真正消费 `TeamWorkspaceSnapshot`

## Solution Design

- Stage 3 只做主读链切换，不提前推进完整 bridge 显式化与页面热点重写。
- 最小实施顺序：
  1. 先补 TeamWorkspaceSnapshot 与最小 Team 子域读接口
  2. 再补 web-client 对应 read contract
  3. 最后切 TeamPage 的主读链，让 workspace-first 容器优先消费 snapshot
- 本阶段不碰：
  - 办公室画布深化
  - bridge/import 全面显式化
  - interaction-agent 主流程

## Complexity Assessment

- Atomic steps: 5+（gateway / web-client / web 主读链 / 验证 / git） → +2
- Parallel streams: yes（gateway/client/frontend 读模型切口探索可并行） → +2
- Modules/systems/services: 3+（gateway / web-client / web） → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 读模型切换涉及 Team 主状态与页面主读链，且必须和 Stage 1/2 保持最小 blast radius，因此需要独立工作流与运行态总表。
