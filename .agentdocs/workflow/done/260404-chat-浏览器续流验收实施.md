# 260404 Chat 浏览器续流验收实施

## Task Overview

在 gateway 级 verification 已覆盖 attach 恢复主链后，继续补一条浏览器层自动化验证，覆盖页面刷新后恢复中的流内容、运行态提示与停止控制。

## Current Analysis

- 现有保障包含 gateway route tests、gateway verification、hook tests 与 ChatPage 回归。
- 还缺一条更接近用户真实刷新路径的浏览器层验证。
- `apps/web/e2e` 已有 Playwright 基础设施，且现有用例大量使用 `page.route()` mock gateway 响应。

## Solution Design

- 新增单条 Chromium 优先的 Playwright e2e：
  - 预写 auth-store / onboarding / telemetry storage
  - 模拟页面刷新前遗留的 `sessionStorage active stream snapshot`
  - mock `/sessions/:id`、`/stream/active`、`/stream/attach`
  - 验证刷新后页面仍显示恢复中的内容、运行态提示和停止控制
- 本轮浏览器层稳定覆盖的是**用户可见恢复体验**；attach 协议细节继续由 gateway verification、hook tests 与 route tests 保证。

## Complexity Assessment

- Atomic steps: 3–4 → 0
- Parallel streams: e2e 用例 + 本地运行 + 记录同步 → +2
- Modules/systems/services: web + playwright infra → +0
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 3
- **Chosen mode**: Full orchestration
- **Routing rationale**: 尽管主要改动在 web e2e，但需要跑真实浏览器并同步归档记录，适合作为单独已完成工作流沉淀。
