# Agent 实体管理升级方案

## Task Overview

把现有 `/agents` 页面从“用户级偏好管理”升级为真正的 Agent 实体管理台，支持编辑 Agent 本体、添加自定义 Agent、禁用/移除、以及单个/全部恢复默认配置。

## Current Analysis

- 现有 `/agents` 页面只管理 `agent_preferences`，可编辑显示名/备注/收藏/隐藏，但不能修改 Agent 本体，更不能新增/移除。
- 当前 `services/agent-gateway/src/routes/capabilities.ts` 里的 Agent 列表仍来自硬编码 `BUILTIN_AGENT_DEFINITIONS`，没有用户级实体存储层。
- 现有系统里，Agent id/label/description/canonicalRole/aliases 主要被 `/capabilities`、Chat slash 建议和 `buildCapabilityContext()` 消费；没有发现必须依赖硬编码 label 的执行链路。
- 因此可行的最小升级路径是：新增 **用户级 Agent Catalog**（覆盖 builtin + 承载 custom agent），并让 `/capabilities` 只消费“当前启用 Agent”的聚合结果。

## Solution Design

### 目标范围

本次升级必须支持：

1. **编辑 Agent 本体**：至少可编辑 `label`、`description`、`aliases`、`canonicalRole`、`enabled`。
2. **添加自定义 Agent**：用户可新增 custom agent。
3. **禁用与移除**：所有 Agent 可禁用；custom agent 可移除；builtin agent 不移除实体，只允许恢复默认。
4. **恢复默认**：支持单个 Agent 恢复默认，以及全部恢复默认。

### 存储模型

- 新增用户级 `agent_catalog`（先落在 `user_settings`，避免引入新表），内部包含：
  - `builtinOverrides`：按 builtin agent id 存用户覆盖值
  - `customAgents`：用户新增的完整 custom agent 实体
- builtin 默认值仍来自 gateway 静态基线定义；恢复默认时移除 override 即可。
- custom agent 创建时记录 `defaultDefinition`，用于单个/全部 reset。

### API 目标

- `GET /agents`：返回完整 Agent 实体列表（builtin + custom，含 origin / enabled / removable / resettable / currentDefinition）
- `POST /agents`：新增 custom agent
- `PUT /agents/:agentId`：更新实体
- `DELETE /agents/:agentId`：移除 custom agent
- `POST /agents/:agentId/reset`：单个恢复默认
- `POST /agents/reset-all`：全部恢复默认

### Capabilities 集成原则

- `/capabilities` 继续作为统一能力目录，但其 `kind === 'agent'` 子集改为来自 `GET /agents` 同源的“启用中 Agent 实体”。
- 已禁用的 Agent 不再出现在 `/capabilities`；因此会同步影响 Chat slash 建议和 `buildCapabilityContext()`。

- Oracle 结论：当前升级已满足“编辑 Agent 本体 / 添加 / 禁用 / 移除 / 单个与全部恢复默认”的最小真实实现范围，无阻塞缺口。
