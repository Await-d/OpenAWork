# 260405 Chat 真实端到端停止验证实施

## Task Overview

在真实 gateway 驱动的刷新恢复 E2E 已打通后，继续补上“刷新后 attach 恢复成功时，当前页仍可直接停止并正确收口到稳定终态”的真实浏览器验证。

## Current Analysis

- 当前真实 E2E 已覆盖：发送消息 → 第一段流文本出现 → 刷新 → attach 恢复 → 最终消息落定。
- 仍缺一个真实控制边界：刷新后虽然恢复了流，但当前页是否还能真正 stop，并把 UI 收口到稳定状态。
- 在实现过程中发现一个真实 bug：`useGatewayClient` 的 active request 句柄只存在 ref 中，句柄失效后不会触发 React 重渲染，导致 UI 仍误显示 precise control。

## Solution Design

- 复用现有 `chat-stream-attach-live.spec.ts` 与 live config；
- 新增第二条真实 E2E：
  - 发送消息
  - 第一段流文本出现后刷新
  - 刷新后点击停止
  - 断言运行提示消失，后续第二段文本不再出现
- 修复 `useGatewayClient`：
  - active request 句柄改为 ref + React state 同步
  - 当句柄失效时，页面能及时从 precise 降级到 best_effort / none
