# .agentdocs/workflow/done/260330-发布前质量门禁补齐方案.md

## Task Overview

根据上一轮自动化发布与构建检查结果，在不处理当前开发中的 P0 红项（Desktop 前端独立构建、Gateway sidecar 二进制构建）的前提下，为现有发布链补齐 P1 级发布前质量门禁。

## Current Analysis

- `prepare-release.yml` 原先直接 bump、提交、打 tag / dispatch，没有任何发布前 gate。
- 当前本地已验证为绿色的检查包括：`pnpm format:check`、`pnpm typecheck`、`pnpm --filter @openAwork/web build`、`pnpm --filter @openAwork/agent-gateway test:unit`、`pnpm --filter @openAwork/mobile test`、`scripts/release-notes.mjs ... --dry-run`。
- 已知 P0 红项：`pnpm --filter @openAwork/desktop exec vite build`、`pnpm --filter @openAwork/agent-gateway build:binary`，用户明确要求先不要纳入门禁。
- 额外实测红项：CI 过滤集合中的 lint 当前会被 `packages/lsp-client` 现有 `@typescript-eslint/no-redundant-type-constituents` 错误卡住，因此本次不纳入 P1 门禁。

## Solution Design

- 在 `prepare-release.yml` 的 `Install dependencies` 之后、版本 bump 之前增加发布前门禁。
- 当前仅接入已验证绿色且对发布有效的检查：格式检查、全量类型检查、Web 生产构建、Gateway 单元测试、Mobile 单元测试、发布稿 dry-run 校验。
- 文档明确声明：Desktop/Gateway 两个开发中 P0 构建红项，以及当前 lint 红项，不属于这次门禁范围。
