# Huddy / Buddy 伴侣图标动画迁移

## Task Overview
把 `temp/claude-code-sourcemap/restored-src/src/buddy/` 中的 companion 视觉层迁入 OpenAWork Web：不仅保留当前 active companion 的动态展示，还把源仓中的全部 companion 物种、sprite 帧和气泡/宠爱动画以可视化方式呈现在界面中。

## Current Analysis
- OpenAWork 在本轮前已具备 active companion 的 ASCII sprite 和基础 idle / pet hearts 动画，但界面中没有“全部伴侣图鉴”的可视化入口。
- `temp/claude-code-sourcemap` 的 `src/buddy/` 提供的核心资产不是独立图片，而是 **deterministic species + 多帧 sprite + 说话气泡 + pet hearts + `/buddy` 触发语义**。
- 因此迁移重点落在 Web 视觉组件，而不是单纯搬运 SVG/PNG。

## Solution Design
- 通过 `companion-sprite-model.ts` 增加按 species 生成预览骨架的能力，支持图鉴中稳定渲染全部 18 个物种。
- 新增 `companion-visual-showcase.tsx`，统一 active companion hero 与全部图鉴卡片的动画渲染逻辑。
- Settings Buddy tab 接入 visual showcase，形成“当前 companion + 全部伴侣图鉴”双层展示。
- 后续增强已补齐 `companion-visual-metadata.ts`：在不扩展功能面的前提下，为图鉴与主舞台增加 rarity color、deterministic stats、当前物种高亮和 `/buddy` 彩虹触发强调，解决“图鉴会动但身份语义偏弱”的缺口。

## Complexity Assessment
- 原子步骤：5 个核心步骤（源仓提取 / 视觉组件抽象 / Settings 图鉴 / active companion 升级 / 测试验证） → +2
- 并行流：源仓抽取与 OpenAWork 挂点分析并行 → +2
- 模块/系统/服务：`apps/web/components` + `apps/web/pages/settings` + `.agentdocs` → +1
- 单步 >5min：视觉组件抽象与回归验证超过 5 分钟 → +1
- 持久化审查产物：需同步 workflow / runtime / index → +1
- OpenCode 可用：→ -1
- **总分：6**
- **选定模式：Full orchestration**
- **路由理由：这是一个跨多个 Web 组件和设计层的真实迁移任务，需要保存迁移范围、视觉落点与验证证据。**

## Success Criteria
- OpenAWork 界面中可以可视化展示全部 companion 物种，而不是只有当前 active companion。
- 每个 companion 预览具有真实动画（idle 多帧 + pet hearts），而不是静态文本列表。
- 当前 active companion persona 区具备更完整的视觉呈现，并与图鉴共享渲染逻辑。
- 相关 Web 组件测试、包级 typecheck、Web build 通过。
