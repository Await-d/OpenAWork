# .agentdocs/workflow/done/260330-统一系统版本来源方案.md

## Task Overview

统一当前仓库中分裂的系统版本来源。目标是让 Web、Desktop、Gateway、Mobile 都遵循同一套版本规则，并明确“构建期动态版本”和“运行期可查询版本”的边界。

## Current Analysis

- Web 左下角角标使用 `scripts/version.mjs` + `scripts/vite-plugin-version.mjs` 注入的 `__APP_VERSION__`
- Desktop 在 `apps/desktop/vite.config.ts` 内联了一套重复的 git 版本拼接逻辑
- Gateway `GET /settings/version` 原先读取 `process.env['npm_package_version']`
- Mobile Settings 页面原先直接硬编码 `OpenAWork Mobile v0.0.1`
- Desktop Tauri 打包版本与前端显示版本需要对齐到同一根版本源

## Solution Design

- 将 `scripts/version.mjs` 作为统一版本规则中心：用户可见版本使用根 `package.json` 的 semver，git 元数据保留为 `buildVersion`
- Web 与 Desktop 共用 `scripts/vite-plugin-version.mjs`
- Gateway 改为基于 `process.cwd()` 的根包优先、当前包兜底版本发现，不再依赖脆弱的源文件相对路径
- Mobile 改为 `app.config.ts` 从根版本注入，再通过 `expo-constants` 在界面读取，移除运行时代码跨级导入根 `package.json`
- Tauri `tauri.conf.json` 的版本改为指向根 `package.json`
