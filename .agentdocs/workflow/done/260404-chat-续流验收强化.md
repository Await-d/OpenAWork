# 260404 Chat 续流验收强化

## Task Overview

在 Chat 真正续流 Phase 2A / 2B 已完成的基础上，补一层更接近真实场景的自动化验收，把 active 查询、attach、回放、live 续流与终态收口串成 gateway verification。

## Current Analysis

- 现有保障主要来自 route 单测、hook 测试与 ChatPage 回归。
- 这些测试足以验证协议与页面行为，但缺少一条使用真实 Fastify + in-memory DB 的 gateway 验收脚本。
- `services/agent-gateway/src/verification/` 已有同类验证入口，适合把 attach 恢复纳入 durable 验收链。

## Solution Design

- 新增 `src/verification/verify-stream-attach-recovery.ts`
- 使用真实 gateway 路由：
  - `GET /sessions/:id/stream/active`
  - `GET /sessions/:id/stream/attach`
- 直接种入：
  - `session_runtime_threads`
  - `session_run_events`
- 验证 attach 路径会先回放 durable `seq=4`，再接上 live `seq=5(done)`
- 将该脚本并入 `test:durable`

## Complexity Assessment

- Atomic steps: 3–4 → 0
- Parallel streams: verification 脚本 + package script + build/test → +2
- Modules/systems/services: 2 → +0
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 3
- **Chosen mode**: Full orchestration
- **Routing rationale**: 虽然改动面不大，但涉及验证体系接线、脚本落库和最终记录沉淀，需要作为独立已完成工作流保留。
