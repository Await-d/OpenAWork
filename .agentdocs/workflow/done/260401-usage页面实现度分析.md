# .agentdocs/workflow/260401-usage页面实现度分析.md

## Task Overview

对项目中 usage 相关页面与路由做一次实现度审计，确认当前已完成的前端展示、后端数据源、记录/持久化/埋点链路，以及明显的未完成点或空壳实现。

## Current Analysis

- 用户关注点不是单纯“页面能不能打开”，而是 **usage 页面里的功能是否实际上没做完，也没有被记录下来**。
- 初步定位到两条前端入口：`apps/web/src/pages/UsagePage.tsx` 与 `apps/web/src/pages/settings/usage-tab-content.tsx`。
- 初步定位到两条后端路由：`GET /usage/records` 与 `GET /usage/breakdown`，以及 SQLite `usage_records` 表。
- 分析重点需要拆成三层：UI 展示是否存在、后端读取是否存在、写入/明细记录是否真实打通。

## Solution Design

1. 先确认 usage 的页面入口、路由挂载与测试覆盖。
2. 再确认 usage 数据读取链路与数据库结构。
3. 最后专门核查“记录/持久化/埋点/明细拆分”是否存在写入端、细粒度来源与真实事实源。

## Notes

- 当前任务只做项目分析，不改业务代码。
- 已确认两条前端入口：独立 `/usage` 页面与 `SettingsPage` 内 `usage` 标签页；两者都直接请求 `/usage/records`、`/usage/breakdown`，并额外读取 `/settings/model-prices`。
- 已确认后端存在 `usage_records` 表与读取路由，但 `usage/breakdown` 当前固定返回空 `breakdown` 数组，明细拆分逻辑尚未实现。
- `settings/model-prices` 当前返回内建静态价格列表；`UsageTabContent` 把 `ModelPriceConfig` 的 `onUpdate` 传成空函数，前端可编辑外观并不会真实持久化。
- 综合结论：usage 页面并非“完全没做”，但目前仍以读取展示为主；核心缺口在于 `usage_records` 无写入链路、breakdown 明细未落地、provider 维度统计缺失、模型价格编辑未接保存、usage 专项 telemetry 未见实现。
- Memory sync: completed
