# .agentdocs/workflow/260403-变更记录功能收口实施.md

## Task Overview

收口 OpenAWork 对话级文件变更记录在客户端与 Web 聊天界面的未完成部分。执行过程中确认 `web-client` API、聊天内 `modified_files_summary`、以及 shared-ui 基础支持已在当前工作树完成，因此实际实施重点收敛为：为 `SessionsPage` 增加会话级 file changes / snapshots / restore 产品入口，并同步完成测试与验证。

## Current Analysis

- 后端 durable 主链已存在，`services/agent-gateway/src/routes/sessions.ts` 已暴露 file changes / snapshots / restore 相关能力。
- 复查当前工作树后确认：`packages/web-client/src/sessions.ts` 已补齐 file changes / snapshots / restore API，聊天内 `modified_files_summary` 也已接通，`packages/shared-ui/src/ChatMessage.tsx` 已提供基础摘要支持。
- 真正尚未收口的是 `apps/web/src/pages/SessionsPage.tsx`：虽然已存在工作区 review 面板，但还没有消费会话级 file changes / snapshots / restore 能力。
- 现有失败点以测试断言漂移和 UI 未接线为主，不涉及后端主链重做。

## Solution Design

- 将 `SessionsPage.tsx` 中超长的 `SessionCard` / `DetailPanel` 拆出到 `apps/web/src/pages/sessions-page/`，先把页面文件压回仓库行数约束内。
- 在新的详情面板里接入 `getFileChangesReadModel`、`compareSnapshots`、`previewRestore` 与 `applyRestore`，提供最小但完整的会话级快照查看、对比、预览与恢复入口。
- 保留既有 workspace review 面板，确保工作区审阅与会话级 durable 快照并存。
- 最后用页面测试、gateway 测试、LSP、TypeScript 与 Web build 形成验收闭环。
