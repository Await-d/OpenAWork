# 260321 Chat 模式会话绑定方案

## Task Overview

用户要求在聊天界面继续完成两件事：

1. 顶部右上角新增 **YOLO 模式**按钮；
2. 将 **对话模式（澄清 / 编程 / 程序员）** 与 **YOLO 状态** 都和当前会话绑定。

绑定的含义是：切换到某个会话时，聊天顶部应该恢复该会话保存过的模式状态，而不是使用页面级临时状态。

## Current Analysis

- `services/agent-gateway/src/routes/sessions.ts` 已有 `metadata_json`，并已经通过 `/sessions`、`/sessions/:id` 返回会话元数据；现有 `/sessions/:id/workspace` 已证明 metadata merge 更新路径可行。
- `packages/web-client/src/sessions.ts` 只支持 list/create/get/delete/rename/import，尚无 metadata 更新接口。
- `apps/web/src/pages/ChatPage.tsx` 当前对话模式 `dialogueMode` 仍是本地 `useState`，切换会话时不会恢复；YOLO 按钮尚不存在。

## Solution Design

### 最小实现原则

1. 不引入新表，继续使用 `sessions.metadata_json`。
2. metadata 新增两个字段：
   - `dialogueMode: 'clarify' | 'coding' | 'programmer'`
   - `yoloMode: boolean`
3. Web client 只新增一个通用 metadata patch API，不重复发明多个专用接口。
4. ChatPage 在以下时机同步状态：
   - 打开已有会话时：从 session metadata 恢复
   - 创建新会话后：把当前模式写入 metadata
   - 用户切换 dialogue / yolo 时：即时 patch 当前 session metadata

## Complexity Assessment

- Atomic steps: 6+ → +2
- Parallel streams: backend metadata API + web client + chat UI/testing → +2
- Modules/systems/services: gateway + web-client + web chat + docs → +1
- Long step (>5 min): yes → +1
- Persisted review artifacts: yes → +1
- OpenCode available: yes → -1
- **Total score**: 6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 这是一个跨 session 存储、客户端 API、聊天 UI 和测试链路的改动，必须用文档和状态表跟踪，否则很容易只修一半。

## Acceptance Criteria

- 聊天顶部右上角出现 YOLO 模式按钮

- 切换任一模式后刷新或再次打开该会话，状态保持不丢失
- 现有 `workingDirectory` metadata 不受影响
