# .agentdocs/workflow/done/260330-自动版本调整方案.md

## Task Overview

将当前“人工修改 semver，再手动触发发布”的版本流程改造成自动调整方案。目标是在保留现有 desktop/mobile 发布工作流的前提下，新增一条自动准备发布的链路：自动推断 bump 级别、统一更新版本文件、提交变更并触发现有发布流程。

## Current Analysis

- 现有发布链由 `.github/workflows/release-desktop.yml`、`release-mobile.yml` 驱动，主要通过 tag 触发
- 现有版本真相源是根 `package.json`，但 monorepo 内 20+ 个 `package.json`、`apps/mobile/app.json` 和 `apps/desktop/src-tauri/Cargo.toml` 仍各自保存版本字段
- desktop preview/stable 与 mobile preview/production 的触发方式不一致：desktop 适合 tag 触发，mobile preview 更适合 workflow_dispatch
- 仓库 commitlint 已收敛 conventional commit 类型，可用于自动推断 major/minor/patch

## Solution Design

- 统一使用 `scripts/release-version.mjs` 作为版本 bump 单入口，支持 `auto|patch|minor|major` 与显式版本输入
- 新增 `prepare-release.yml`，自动安装依赖、运行版本脚本、提交 bump 结果、然后按目标触发现有 desktop/mobile 发布工作流
- `auto` 模式基于最近 release tag 以来的 conventional commits 自动推断 bump 级别：breaking→major，feat→minor，其余有效提交→patch
- 版本号采用 9 进 1 进位规则：例如 `0.0.9 → 0.1.0`，`0.9.9 → 1.0.0`
- 保持现有 `release-desktop.yml` / `release-mobile.yml` 主体不变，只做必要增强以接收自动化触发
