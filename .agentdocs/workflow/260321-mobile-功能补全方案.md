# 260321 Mobile 功能补全方案

## Task Overview

基于 2026-03-21 代码审计，移动端存在 13 个真实未完成问题，涵盖导航可用性、UI 中文化、代码规范、Onboarding 逻辑等。本方案目标是将所有问题分优先级完整收口。

## Current Analysis

### P0 — 影响可用性（用户无法正常使用）
- M-01：`AppNavigator.tsx` onboarding 分支返回 `null`，OnboardingWizard 根本不渲染
- M-02：Sessions 页面没有入口按钮可进入 Settings
- M-03：Chat 页面没有返回按钮，用户进入后无法回到会话列表

### P1 — 用户体验问题
- M-04：Chat 标题硬编码 `'Chat'`，未显示会话标题
- M-06：Sessions Header 缺少 Settings 导航按钮（`app/sessions.tsx`）
- M-07：NetworkBanner 文字是英文，其他 UI 全是中文
- M-11：`app/_layout.tsx` sessions 路由 header 没有 Settings 按钮

### P2 — 代码规范/质量
- M-05/M-12：`app/chat/[sessionId].tsx` 使用 `as any[]` 类型断言（违反项目规范）
- M-08：OnboardingWizard HostHealthStep 是 mock（2秒 timer），需改为真实 Gateway 健康检查
- M-09：ClientScanStep 配对 JSON 解析后没有保存/连接逻辑
- M-10：CloudLoginStep 完全 mock，需接入真实认证

### P3 — 架构
- M-13：`src/screens/` 与 `app/` 路由存在重复实现，需明确哪套是主路径

## Solution Design

- `app/` 目录是 Expo Router 运行时真相源（`_layout.tsx` 配置了 Stack）
- `src/screens/` 是旧手动状态机路径（`AppNavigator.tsx`），AGENTS.md 禁止替换
- 两套并存属历史遗留，修复只针对 `app/` 路由层
- P0/P1 优先，P2 紧随，P3 最后

## Complexity Assessment

- Atomic steps: 10 → +2
- Parallel streams: P0/P1/P2 部分可并行 → +2
- Modules: app/ screens + navigator + hooks + components → +1
- Long step: 部分 >5min → +1
- Persisted review: yes → +1
- OpenCode available: yes → -1
- **Total score**: +6
- **Chosen mode**: Full orchestration
- **Routing rationale**: 跨多个文件和层次，P0 必须先完成才能验证 P1，需协调顺序执行。
