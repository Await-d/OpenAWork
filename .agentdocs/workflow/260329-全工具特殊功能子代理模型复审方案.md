## 任务概览

再次对照 `temp/opencode` 与 `temp/oh-my-openagent`，复审 OpenAWork 当前仓库中“全部工具定义与可调用面、特殊功能的真实使用方式、以及子代理模型/类别路由”是否一致，并在发现关键偏差时继续收口。

## 当前分析

- 上一轮已经完成 `task` 契约、`background_output/background_cancel`、category prompt append、capabilities 真值与 reference snapshot 路径的一批关键修正。
- 但本轮范围扩大到“所有工具 + 特殊功能 + 子代理模型”，意味着不能只盯 `task` 与 prompt，还要核查 session/history、AST/LSP、MCP wrapper、interactive/background、以及 agent/category 的模型选择与 fallback 心智是否仍有遗漏。
- 本轮重点是确认当前仓库有没有“工具名看起来齐了，但特殊功能或模型路由的用法还不一致”的剩余暗坑。
