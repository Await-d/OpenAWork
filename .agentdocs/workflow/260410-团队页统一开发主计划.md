## 背景与冻结结论

### 为什么要做
- 当前 TeamPage 仍偏“成员 / 任务 / 消息 / 共享会话 / 审计 / presence”的协作页。
- 目标是把它升级成 **多 Agent 运行工作台**，复刻的是参考库的“使用方式”，不是技术结构或视觉皮肤的逐像素复制。

### 学什么 / 不学什么
- **学习**：Supervisor/子 Agent/任务看板/结构化事件流/团队运行闭环。
- **不学习**：参考库的 Electron 主进程中心化结构、README 中未落地的 team 模块边界、全能桌面总控台式实现。

### 已冻结方向（不得反复讨论）
- Web 优先
- 工作区一级
- 总控大盘
- 主对象：`Workspace → Team Instance → Mission Run`
- 执行角色独立于权限成员角色
- 人工消息默认全部先进入 `interaction-agent`
- 设计原则：`upstream-native first / merge-first`
- 画面结构可参考参考库，但**任务动画统一替换为 Buddy/Hubby 动画层**

## 目标与非目标

### 目标
- 让开发者交付一个可运行的 Web Team Runtime 总控页
- 使用现有上游真相源完成并轨：`run events / task projection / session tree / shared session / approval/question / artifacts / agents/capabilities / workflow template`
- 让 TeamPage 成为：
  - 工作区首页入口
  - Team Instance 运行总控
  - 人工介入与共享协作主入口

### 非目标
- 首期不做桌面端/移动端完整总控同步
- 首期不并入自定义 DAG 编辑器
- 首期不做多交互代理或交互/调度双代理模型
- 首期不把 schema 迁移开成独立主线，除非被明确阻塞

## 产品与页面设计冻结稿

### 核心对象模型

#### 1. Workspace
- 一级总控对象
- 对应会话工作区 / 项目
- 首页以工作区卡片或分组展示

#### 2. Team Instance
- 工作区下的一组固定协作编排配置
- 包含：模板来源、执行角色映射、默认 Agent/能力集、上下文策略
- 不等于一次运行

#### 3. Mission Run
- 团队实例上的一次具体执行
- 包含：目标、状态、开始时间、当前阻塞、当前活跃角色、最近事件、关联会话树
- 是 TeamPage 的真正主对象

#### 4. Execution Role
- 如：`interaction-agent / planner / researcher / executor / reviewer`
- 与 `owner/admin/member` 这类权限成员角色分离

#### 5. Shared Context
- 复用共享会话、评论、presence、审批、问题回复等现有链路
- 是 Mission Run 的上下文层，不单独成为主对象

#### 6. Output Surface
- 复用 artifacts、文件上下文、Git 变更、结果摘要
- 是运行结果层

### 页面信息架构

#### 顶部：总控指标卡
- 运行/阻塞/待审批/待回答
- 会话活跃
- 角色负载
- 成本用量
- 变更冲突
- 失败告警

#### 一级：工作区卡片
- 每张卡片展示该工作区的团队实例摘要与最新 Mission Run 状态
- 点击进入工作区总控视图

#### 二级：工作区内 7 个主 Tab
- `总览`：Mission Run 概况、活跃角色、关键状态
- `会话/Agent`：父子会话树、活跃 Agent、运行状态
- `任务看板`：任务推进、状态泳道、阻塞点
- `文件上下文`：共享会话、文件、工作区上下文
- `消息时间线`：事件流、角色消息、审批与提问
- `产物`：artifacts、结果输出、版本切换
- `Git/变更`：文件改动、冲突、提交相关信息

### 统一交互代理
- `interaction-agent` 是团队实例的统一人工入口
- 人工消息默认全部先进入该角色
- 首期职责仅限：**理解并改写需求**
- 首期不承担：完整拆解、汇总、审批守门

### 动画表现冻结约束
- 页面结构、分区方式、Tab 组织、总控布局可强对齐参考库
- 参考库原任务动画**不复用**
- TeamPage / 团队运行界面中的任务动画统一替换为系统内 **Buddy/Hubby 动画表现层**
- Buddy/Hubby 默认只承担：
  - 任务运行中提示
  - 阻塞 / 待审批 / 待回答提示
  - `interaction-agent` 的轻量视觉代理
- Buddy/Hubby 不承担：
  - 替代 DAG / 时间线主视图
  - 逐条复述消息流或任务明细

## 上游能力并轨映射

### 1. 运行事件 → 消息时间线 / 总览状态
- 来源：`run events`
- 落点：顶部状态卡、`消息时间线`、Mission Run 当前阶段/最近阻塞/最近恢复动作

### 2. 任务投影 → 任务看板
- 来源：`/sessions/:id/tasks` + `task_update`
- 落点：`任务看板`、角色负载统计、阻塞列表

### 3. 子会话树 → 会话/Agent Tab
- 来源：`session_child`、parent/child session
- 落点：`会话/Agent`、活跃 Agent 列表、子 Agent 层级

### 4. 共享会话链路 → 文件上下文 / 右侧协作栏
- 来源：shared session、comment、presence、operate、question/permission reply
- 落点：`文件上下文`、右侧审批/共享会话面板、人工进入上下文并继续协作

### 5. Workflow 模板 → Team Instance 创建入口
- 来源：workflow/template 能力
- 落点：从模板创建团队实例，把模板变成实例初始角色/任务配置来源

### 6. Agent 目录 / Capabilities → 执行角色绑定面板
- 来源：`/agents`、`/capabilities`
- 落点：执行角色映射、角色默认 Agent / 能力集

### 7. Artifacts → 产物 Tab
- 来源：artifacts、版本、回退能力
- 落点：`产物`、接受/查看/切换结果版本

### 8. 文件变更 / Git → Git/变更 Tab
- 来源：文件变更、Git 状态、冲突信息
- 落点：`Git/变更`、改动热区与冲突视图

## MVP / 非 MVP

### MVP（首期必须做）
- Web 端工作区一级总控首页
- 工作区内 Team Instance / Mission Run 总控视图
- 顶部指标卡
- 7 个主区 Tab
- 统一交互代理入口
- 审批/恢复侧栏
- 基于现有上游能力的并轨读模型
- Buddy/Hubby 任务动画替代参考库原任务动画

### 非 MVP（后续再做）
- 桌面端完整总控页同步落地
- 移动端总控与全量交互
- 交互代理自动拆解 / 自动汇总 / 审批守门
- 多交互代理或交互/调度双代理模型
- 自定义 DAG 编辑器并入 TeamPage
- 高级 Git/worktree 深编排
- 跨工作区联动大盘

## 风险清单

1. **按 Tab 平分导致热点冲突**：必须按真相源边界拆，而不是按 Tab 平均拆。
2. **DTO 未冻结就接 UI**：A 先接真实字段，B 再改合同，会出现高频返工。
3. **共享会话与运行事件语义漂移**：B 同时碰 team/shared/session-run-events，必须保持 operate / 审计语义一致。
4. **shared-ui 过早卷入**：非必要不提前改通用组件，避免波及 ChatPage。
5. **默认开 schema 主线**：当前已有上游支持，除非被阻塞，否则不把 schema 迁移作为主任务。
6. **Buddy/Hubby 语义越位**：动画层只做状态与交互代理的辅助表达，不替代 DAG/时间线主视图。
