# .agentdocs/workflow/260319-skills市场完整方案.md

## 任务概览

设计完整的 Skills 市场系统，支持：
- 官方策划市场（OpenWork 官方维护）
- **用户自定义第三方市场地址**（团队私有市场、社区市场）
- GitHub 直接安装
- 本地 Skill 开发与加载
- 完整的安装生命周期（下载→验证→权限确认→注册→激活）

## 核心设计：多源注册中心

### 市场源（Registry Source）模型

```typescript
// packages/skill-registry/src/source.ts

export type RegistrySourceType =
  | 'official'   // OpenWork 官方市场（内置，不可删除）
  | 'community'  // 社区维护市场
  | 'private'    // 团队/企业私有市场
  | 'github'     // GitHub 仓库直接安装
  | 'local';     // 本地目录（开发模式）

export interface RegistrySource {
  id: string;             // 唯一标识，如 'official' / 'myteam-registry'
  name: string;           // 显示名称
  type: RegistrySourceType;
  url: string;            // 注册中心 API 基础 URL
  enabled: boolean;       // 是否启用
  priority: number;       // 搜索结果排序优先级（越小越靠前）
  // 认证（私有市场需要）
  auth?: RegistryAuth;
  // 信任设置
  trust: RegistryTrustLevel;
  addedAt: number;
  addedBy: 'system' | 'user';
}

export type RegistryTrustLevel =
  | 'full'      // 官方市场：无需用户确认即可安装 verified skill
  | 'standard'  // 社区市场：安装前展示权限清单，用户确认
  | 'restricted'; // 私有/不知名市场：每次安装需二次确认

export interface RegistryAuth {
  type: 'none' | 'token' | 'basic' | 'oauth2';
  token?: string;          // Bearer token
  username?: string;       // Basic auth
  password?: string;       // Basic auth（存储在 SecureStore）
  oauthConfig?: OAuthConfig;
}

export interface OAuthConfig {
  clientId: string;
  authorizationUrl: string;
  tokenUrl: string;
  scopes: string[];
}
```

## 用户自定义市场源

### 添加方式（三种）

**方式 1：UI 添加（推荐）**
```
设置页 → Skills → 市场源管理 → 「+ 添加市场」
  → 输入 URL
  → 系统自动获取 registry-info.json
  → 显示市场名称、描述、维护者、Skill 数量
  → 选择信任级别（standard / restricted）
  → 可选：输入认证 Token
  → 确认添加
```

**方式 2：配置文件**
```json
// .openwork/config.json 或 ~/.config/openwork/config.json
{
  "skill_registries": [
    {
      "id": "myteam",
      "name": "MyTeam Internal Skills",
      "url": "https://skills.mycompany.internal/v1",
      "auth": {
        "type": "token",
        "token": "${MYTEAM_REGISTRY_TOKEN}"  // 支持环境变量引用
      },
      "trust": "standard"
    },
    {
      "id": "community-hub",
      "name": "Community Skills Hub",
      "url": "https://hub.agentskills.io/v1",
      "trust": "standard"
    }
  ]
}
```

**方式 3：命令行**
```bash
opkg registry add https://skills.mycompany.internal/v1 --name "MyTeam" --token $TOKEN
opkg registry add https://hub.agentskills.io/v1 --name "Community Hub"
opkg registry list
opkg registry remove myteam
opkg registry enable myteam
opkg registry disable community-hub
```

## SkillRegistry 客户端完整接口

```typescript
// packages/skill-registry/src/index.ts

export interface SkillRegistryClient {
  // ── 市场源管理 ──────────────────────────────

  // 获取所有已配置的市场源
  listSources(): Promise<RegistrySource[]>;
  // 添加第三方市场源（UI/CLI 调用）
  addSource(config: AddSourceConfig): Promise<RegistrySource>;
  // 移除市场源（内置官方源不可移除）
  removeSource(sourceId: string): Promise<void>;
  // 启用/禁用市场源
  toggleSource(sourceId: string, enabled: boolean): Promise<void>;
  // 验证市场源是否可达（获取 registry-info.json）
  verifySource(url: string, auth?: RegistryAuth): Promise<RegistryInfo>;

  // ── Skill 搜索与浏览 ────────────────────────

  // 跨所有已启用市场源搜索（按 priority 排序，去重）
  search(query: string, options?: SearchOptions): Promise<SkillSearchResult[]>;
  // 获取 Skill 详情（含 README、截图、更新日志）
  getDetail(id: string, sourceId?: string): Promise<SkillDetail>;
  // 获取分类列表
  listCategories(sourceId?: string): Promise<SkillCategory[]>;
  // 获取策划精选列表（Featured）
  getFeatured(sourceId?: string): Promise<SkillEntry[]>;

  // ── 安装管理 ────────────────────────────────

  // 安装 Skill（含完整生命周期）
  install(id: string, options?: InstallOptions): Promise<InstalledSkill>;
  // 从 GitHub 安装（格式：user/repo 或 user/repo@tag）
  installFromGitHub(repo: string, options?: InstallOptions): Promise<InstalledSkill>;
  // 从本地目录加载（开发模式）
  installFromLocal(dirPath: string): Promise<InstalledSkill>;
  // 卸载 Skill
  uninstall(id: string): Promise<void>;
  // 更新 Skill
  update(id: string): Promise<InstalledSkill>;
  // 批量更新所有有更新的 Skill
  updateAll(): Promise<{ updated: string[]; failed: string[] }>;

  // ── 已安装 Skill 管理 ───────────────────────

  // 列出已安装的 Skill
  listInstalled(): Promise<InstalledSkill[]>;
  // 检查是否有可用更新
  checkUpdates(): Promise<UpdateAvailable[]>;
  // 获取 Skill 当前状态（active/inactive/error）
  getStatus(id: string): Promise<SkillStatus>;
  // 启用/禁用已安装的 Skill
  toggleInstalled(id: string, enabled: boolean): Promise<void>;
  // 查看 Skill 配置
  getConfig(id: string): Promise<Record<string, unknown>>;
  // 更新 Skill 配置
  updateConfig(id: string, config: Record<string, unknown>): Promise<void>;

  // ── 开发者功能 ──────────────────────────────

  // 发布 Skill 到指定市场
  publish(dirPath: string, targetSourceId: string): Promise<void>;
  // 验证 skill.yaml 格式
  validate(dirPath: string): Promise<ValidationResult>;
}

export interface SearchOptions {
  sourceIds?: string[];         // 限定搜索的市场源（空=全部启用源）
  category?: string;
  tags?: string[];
  onlyVerified?: boolean;
  page?: number;
  pageSize?: number;
}

export interface InstallOptions {
  sourceId?: string;            // 指定从哪个市场源安装
  version?: string;             // 指定版本（默认最新稳定版）
  skipPermissionConfirm?: boolean; // 仅在 trust='full' 且 verified=true 时生效
}
```

## 安装生命周期

```
install(id, options)
  │
  ▼
① 解析来源
  │  确定从哪个 sourceId 安装
  │  GitHub 安装：克隆/下载 zip
  │  本地安装：直接读取目录
  ▼
② 获取 manifest
  │  解析 skill.yaml
  │  验证 apiVersion / id / permissions 字段
  │  检查平台兼容性
  │  检查 App 版本要求
  ▼
③ 签名验证（trust != 'local'）
  │  official + verified: 强制验签，失败则中止
  │  standard: 验签，失败显示警告但允许继续
  │  restricted: 验签，失败需要用户二次确认
  ▼
④ 依赖检查
  │  检查 Skill 依赖是否已安装
  │  版本冲突检测（同一 MCP server 不同版本）
  │  有冲突 → 展示冲突详情，让用户选择解决方案
  ▼
⑤ 权限确认（UI 交互）
  │  展示 manifest.permissions 清单
  │  trust='full' + verified → 自动通过（无需确认）
  │  trust='standard' → 展示权限列表，用户点击「安装」确认
  │  trust='restricted' → 展示权限列表 + 警告横幅，需输入确认文字
  │  high-risk 权限（filesystem write / camera / location）→ 单独高亮
  ▼
⑥ 下载与验证
  │  从 registry URL 下载 skill 包（.agentskill 格式，zip）
  │  校验 SHA256 哈希
  │  验证包签名（Ed25519）
  ▼
⑦ 安装到本地
  │  解压到 ~/.config/openwork/skills/{skill-id}/
  │  写入 installed.json（记录版本、来源、安装时间）
  ▼
⑧ 注册到 ToolRegistry
  │  解析 skill.yaml 中的 tools 定义
  │  调用 ToolRegistry.register(skillTools)
  │  MCP 绑定：按 lifecycle.activation 决定是否立即启动
  ▼
⑨ 写入 AuditLog
  │  记录：{ type: 'skill_install', skillId, version, sourceId, timestamp }
  ▼
✅ 完成
  │  通知 UI 刷新已安装列表
  │  显示 Toast：「已安装 {displayName}」

## 第三方市场 API 规范（供市场开发者参考）

第三方市场需实现以下 REST API（与官方市场相同协议）：

```
GET  /v1/registry-info.json          → RegistryInfo（市场元信息）
GET  /v1/skills?q=&category=&page=   → { skills: SkillEntry[], total: number }
GET  /v1/skills/{id}                 → SkillDetail
GET  /v1/skills/{id}/versions        → SkillVersion[]
GET  /v1/skills/{id}/download?v=     → .agentskill 文件流
GET  /v1/categories                  → SkillCategory[]
GET  /v1/featured                    → SkillEntry[]
POST /v1/publish                     → 发布新 Skill（需认证）
GET  /v1/updates?ids=id1,id2,...     → UpdateAvailable[]（批量检查更新）
```

**私有市场认证**：在请求 Header 中携带 `Authorization: Bearer {token}`。

**最小实现**（只需 3 个接口即可被客户端识别）：
- `GET /v1/registry-info.json`
- `GET /v1/skills`
- `GET /v1/skills/{id}/download`

## 安全机制

### 签名验证

```typescript
// 官方市场：Ed25519 签名，公钥内置
const OFFICIAL_PUBLIC_KEY = 'ed25519:...';

// 第三方市场：从 registry-info.json 中获取公钥
// {
//   "signingPublicKey": "ed25519:..."
// }

// 每个 .agentskill 包含 manifest.sig 签名文件
// 安装时验证：sign(skill.yaml + sha256(bundle)) == manifest.sig
```

### 恶意包防护

| 防护层 | 机制 |
|--------|------|
| 官方验证 | `verified: true` = 人工审核通过，签名验证 |
| 权限清单 | 安装前展示完整权限，用户知情同意 |
| 来源白名单 | trust='restricted' 需二次确认 |
| 沙箱执行 | Skill 工具调用受 ToolRegistry 权限限制 |
| 审计日志 | 所有安装/卸载/调用记录到 AuditLog |
| 隔离存储 | 每个 Skill 独立目录，不可访问其他 Skill 文件 |

### disabled_tools 防护

用户可在设置中将特定 Skill 的工具完全从 LLM 视野中移除：
```typescript
// PermissionManager.disableTool('skill:brave-search:search')
// → ToolRegistry 中该工具标记为 hidden=true
// → 不出现在 LLM 的 tool list 中（防 prompt injection）
```
